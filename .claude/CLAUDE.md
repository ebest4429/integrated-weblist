# CLAUDE.md

## 세션 시작

> SessionStart 훅이 세션 시작 시 아래 3개 파일 읽기를 강제한다
> Bash/Edit/Write는 3개 파일 Read 완료 전까지 기술적으로 차단된다

강제 읽기 순서:
1. 마스터플랜 (`.claude/plans/*-master.md`)
2. 현재 활성 페이지플랜 — `## 작업 항목` 섹션
3. `.claude/WORKSPACE.md` — `## 현재 위치` + `## 남은 과제`

필요 시 추가로 읽는 파일:
- `.claude/CONTEXT.md` — 프로젝트 구조
- `.claude/RULES.md` — 작업 원칙

## 페이지플랜 규칙

- 모든 페이지플랜의 현재 작업 섹션 이름: `## 작업 항목` (고정)
- 마스터플랜 상단 `**현재 활성 플랜:** \`파일명\`` 으로 활성 플랜 지정
- `plans/` 직하위 = 활성 중 또는 완료된 페이지플랜만
- `plans/designs/` = 아직 활성화되지 않은 사전설계 (미래 Phase 설계는 여기에만)

**Phase 전환 절차 (반드시 이 순서로):**
1. `WORKSPACE.md` → `WORKSPACE.{현재플랜stem}.md` 로 이름 변경 (Bash mv)
2. 새 `WORKSPACE.md` 작성
3. 마스터플랜 활성 플랜 포인터 수정
4. `plans/designs/` 의 사전설계 참조하여 `plans/` 에 새 페이지플랜 Write
   → 훅이 아카이브 여부 자동 검증

**돌발상황:** 활성 플랜 포인터를 뒤로 돌리지 않고 `plans/hotfix-{날짜}-{설명}.md` 임시 플랜 생성

## 프로젝트 정체성

**통합 웹 리소스 허브 (Integrated Web Resource Hub)**
AI·개발·콘텐츠 제작·마케팅·자동화에 필요한 서비스 링크를 분류·통합한 웹 리소스 허브.
**기술 스택: Vite + React + Tailwind CSS** (빌드형. 단일 HTML 아님)
사이드바 네비게이션(그룹핑) + 검색 + 즐겨찾기 구조. 상단 nav 없음.

## 플랜 기록 원칙 (필수)

플랜 파일에 결정사항을 기록할 때:

- **결정사항 + 취지를 함께 기록한다** — 결정만 있고 취지가 없으면 새 세션 AI가 맥락 없이 잘못된 방향으로 해석한다
- 취지 기록 형식: 결정 내용 → `취지:` 또는 주석(`>`)으로 왜 이 결정을 했는지 명시
- 특히 **보류·금지·대기** 항목은 반드시 이유를 함께 기록한다
  - 나쁜 예: `🔲 구글 시트 기록 확인 — 보류`
  - 좋은 예: `🔲 구글 시트 기록 확인 — 보류. 취지: 다수 세션 시도해도 상태 불변. 방향 자체 재검토 필요. 수정 시도 금지.`
- 협의 과정에서 결정된 내용은 협의 당시 AI도 알고 사용자도 알지만, **다음 세션 AI는 모른다**

## 파일 내용 보고 원칙 (필수)

파일을 읽고 내용을 보고하거나 분석할 때:

- **파일에 실제로 있는 내용만 말한다**
- 파일에 없는 내용을 "누락", "추가로", "참고로" 등의 표현으로 덧붙이는 것 **금지**
- 추론·추정·배경지식을 파일 내용인 것처럼 섞어서 말하는 것 **금지**
- 파일 외 정보를 언급해야 할 경우 반드시 **(파일 외 정보)** 라고 명시한 후 사용자 확인을 받는다
- 확인하지 않은 사실을 단정적으로 말하는 것 **금지** — 불확실하면 "확인이 필요합니다"라고 말한다

## WORKSPACE.md 운용 원칙 (필수)

**파일 구조 (3섹션 고정)**
- `## 현재 위치` — 항상 읽음
- `## 남은 과제` — 항상 읽음
- `## 진행 이력` — 필요할 때만 추가 요청

**기본 규칙**
- **Edit 도구로만 수정**: 내용 추가·수정은 자유롭게 허용
- **Write 도구 차단**: 훅이 자동으로 덮어쓰기를 막는다
- **체크박스 없음**: 진행상황은 서술 방식으로 기록. 체크박스는 페이지 플랜에만 존재
- **진행 이력**: 최신→과거 순으로 추가

**새 WORKSPACE.md가 필요한 경우 (새 Phase 시작 등)**
1. 기존 WORKSPACE.md를 이름 변경: `WORKSPACE.{플랜명}.md`
2. 새 WORKSPACE.md 작성 (훅이 이름 변경 확인 후 허용)

**보류 항목**
- 완료되지 않은 항목은 새 WORKSPACE.md에도 보류 섹션으로 계속 포함

**금지**
- 사용자 지시 없이 WORKSPACE.md 내용 삭제 금지
- 체크박스 형식 사용 금지
