> **영구 참조 문서** — Phase가 바뀌어도 삭제하지 않는다.

## 구현 단계

| 단계 | 형태 | 상태 |
|------|------|------|
| **Phase 1** | 프롬프트 구조 설계 (Claude Desktop 테스트 스킵) | ✅ 완료 |
| **Phase 2** | 최소 FastAPI 로컬 서버 + 대화 흐름 검증 | ✅ 완료 (인프라 + 챗봇 골격) |
| **Phase 3** | 협의 항목 결정 + 프롬프트 완성 (FastAPI 로컬 테스트) | 🔄 진행 중 |
| **Phase 4** | FastAPI 완성 → Google Sheets·카카오·Railway | 🔲 대기 |

> Claude Desktop 테스트 방식 폐기 (2026-04-04 결정)
> 이유: 단일 시스템 프롬프트 제약, 파일 분기 불가, FastAPI로 직접 전환이 구조적으로 우월

## 기술 스택 (확정)

| 영역 | 기술 | 결정 이유 |
|------|------|---------|
| 백엔드 | Python + FastAPI | 경량, 빠름 |
| 세션 관리 | 인메모리 (Python dict) | 대화는 5~15분, 재시작 허용 가능. 외부 DB 불필요 |
| AI API | Gemini 2.0 Flash | 무료티어, 단순 수집 흐름에 충분 |
| AI 대안 | Claude Haiku | 환경변수 전환, 품질 미흡 시 교체 |
| 접수 데이터 | Google Sheets | 중개사 직접 확인·관리, 재배포 불필요 |
| 설정 관리 | Google Sheets | 비개발자 직접 수정 가능 |
| 카카오 연동 | 카카오 i 오픈빌더 | 공식 채널, 초기 1회 설정 후 불변 |
| 배포 | Railway ($5/월) | 항상 켜짐, SSL 자동, 단순 관리 |
| 대화 로그 | 없음 (추후 필요 시 추가) | 현재 요구사항 없음 |
| 별도 DB | 없음 | Google Sheets + 인메모리로 충분 |

## 아키텍처

**Phase 2 — 로컬 테스트용 최소 구성:**
```
테스트 클라이언트 (curl / Postman / 간단한 UI)
      ↓
FastAPI 서버 (로컬 uvicorn)
      ├── 인메모리 세션     — Python dict 기반 임시 관리
      ├── Gemini / Claude API — 응답 생성
      └── .Source-Files/매물특징/ — 카테고리별 파일 동적 로드
```

**Phase 4 — 실서비스 구성:**
```
카카오톡 사용자
      ↓
카카오 i 오픈빌더  (초기 1회 설정)
      ↓ webhook POST
FastAPI 서버 (Railway $5/월)
      ├── 인메모리 세션    — 대화 진행 상태 관리
      ├── Gemini / Claude  — 응답 생성 (환경변수 전환)
      └── Google Sheets    — 접수 기록 + 설정 읽기
```

**매물특징 파일 분기 구조 (Phase 2~):**
```
.Source-Files/매물특징/
├── 1-아파트-주상복합-오피스텔-도시형-다세대.md
├── 2-다가구-상가주택-단독주택.md
├── 3-원룸-투룸-쓰리룸.md
├── 4-수익성건물.md
├── 5-구분상가-사무실-지식산업센터-오피스텔.md
├── 6-공장-창고.md
├── 7-토지.md
├── 8-분양권.md
├── 9-재개발-재건축-입주권.md
└── 10-기타.md
```
사용자가 번호 선택 시 해당 파일만 로드 → 시스템 프롬프트에 동적 주입

## 구글 시트 구조

| 시트명 | 용도 | 관리 주체 |
|--------|------|----------|
| [설정] 중개사무소 정보 | 이름·대표자·전화·주소 | 중개사 직접 수정 |
| [설정] 톤앤매너 | 프리셋 선택 (A/B/C/D) | 중개사 직접 수정 |
| [설정] 메시지 | 인사말·안내문구 | 중개사 직접 수정 |
| [데이터] 접수 내역 | 챗봇 접수 완료 데이터 자동 기록 | 자동 (읽기 전용) |

서버 기동 시 설정 시트를 읽어 프롬프트에 반영한다.
설정 변경 시 재배포 불필요 — 시트 수정만으로 즉시 반영된다.

## 카카오 설정 (초기 1회)

1. 카카오 비즈니스 채널 개설
2. 카카오 i 오픈빌더 — 스킬 등록 (webhook URL: Railway 배포 URL)
3. 채널 연결 완료

이후 변경 불필요. 서버 URL 변경 시 webhook URL만 재등록.
