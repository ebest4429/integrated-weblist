- ✅ 시트 구조 분리

    - ※ 매물별 질문 항목 확정 후 컬럼 점진적 보강 예정

### 8-1. 테스트 URL 배포 (item 8 완료 후)

- ✅ 마크다운 렌더링 제거 (kakao.py 후처리 함수 추가)
- ✅ 구글 시트 저장 버그 수정 — extract_request_summary() response.text None 처리
- ✅ 구글 시트 데이터 추출 수정 — response.text 대신 candidates[0].content.parts[0].text 직접 접근
- ✅ 구글 시트 추출 버그 추가 수정 — 코드블록 처리 후 빈 텍스트 json.loads 오류 수정
- 🔲 전체 상담 흐름 완주 확인
- 🔲 테스터에게 Railway 웹 URL 배포 (카카오 채널 또는 브라우저 URL)
- 🔲 테스터 피드백 수집
- 🔲 피드백 반영 후 완성 단계 진입

> 배포 방식: 카카오 채널 + 웹 URL 동시 운용 가능. 테스터에게는 URL 직접 공유.
> 어뷰징 대응: 테스트 기간 중 무제한(TEST_MODE=true). 완성 후 일일 횟수 제한으로 구현.
> 토큰 기반 URL(기간/횟수 제한): 완성 단계 이후 필요시 추가 구현.

> **[구글 시트 기록 확인 분리 — 취지]**
> 구글 시트 기록은 이 시점에서 확인하지 않는다. 다수 세션에서 수정 시도해도 접수ID+접수일시만 기록되고 나머지 공백인 상태가 불변. 나빠지지도 나아지지도 않는다는 것은 부분 수정으로 해결 불가, AI 추출 방향 자체가 잘못됐다는 신호. 질문내용·답변 확정 후 전면 재분석이 필요. **지금 수정 시도 금지.**

### 8-2. 웹링크 토큰 접근 제어

> **배포방식 결정 (2026-04-06):** 주사용방식 웹링크. 카카오 채널은 구현 유지하되 실무 사용 빈도 낮음.

- ✅ 토큰 타입 3종 구현
  - `once` — 1회 사용 후 소진 (고객 정식 접수용)
  - `timed` — 발급 후 20분 유효, 반복 사용 가능 (시연/재상담용)
  - `tester` — 만료 없음 영구 유효 (테스트 기간 테스터 전용)
- ✅ 토큰 발급 엔드포인트 (`POST /token/new?type=once|timed|tester`)
- ✅ 토큰 검증 미들웨어 (`type`, `expires_at`, `used` 필드)
- ✅ 웹 클라이언트 진입 URL: `https://.../?token=xxx`
- ✅ TEST_MODE=true 시 토큰 검증 건너뜀 (테스터 고정 URL 사용)
