### 매물종류 선택지 설계 원칙 (2026-04-05 확정)

- 매물종류 선택지는 모든 거래당사자(매도인·매수인·임대인·임차인)에게 1~10번 동일하게 표시
- 거래당사자 유형에 따른 선택지 제한 금지 — 제한은 선택 이후 질문지 구성 단계에서 처리
- 거래 불가 유형 안내: 매물특징 파일에 해당 거래당사자 열이 없는 경우 AI가 안내
  - 3번(원룸·투룸·쓰리룸): 임대·임차 전용. 매도인·매수인 선택 시 안내
  - 8번(분양권), 9번(재개발·재건축·입주권): 매매 전용. 임대인·임차인 선택 시 안내

### 매물특징 파일 보강 방향 (2026-04-05 확정)

**서브타입 혼재 파일 (상세 자료 제공 시 우선 보강 대상):**

| 파일 | 서브타입 | 현재 상태 |
|------|---------|---------|
| 2번 다가구·상가주택·단독주택 | 상가주택 | `(상가주택인 경우만)` 표기 존재 |
| 5번 구분상가·사무실·지식산업센터·오피스텔 | 지식산업센터 | `(지식산업센터)` 표기 존재 |
| 6번 공장·창고 | 공장 / 창고 | `(공장)` `(창고)` 표기 존재 |

**보강 방법 (구조 변경 없음):**
1. 기존 기본정보/추가정보 테이블에 **행(항목) 추가**만 한다
2. 서브타입 전용 항목은 `(서브타입명)` 표기를 유지한다
   - 예: `호이스트·크레인 | 유/무, 용량 (공장) | 필요 여부`
3. 완성프롬프트.md의 전역 지침이 서브타입 분기를 자동 처리한다
   - "기본정보 항목에 (서브타입) 표시가 있는 경우, 해당 유형 항목만 질문"

**보강 우선순위:**
- 실무 상담 데이터 기반 항목 추가 (Phase 4 배포 후 피드백 반영)
- 세부 유형별 특화 항목 추가 시 기존 표기 방식 그대로 준수

### 챗봇 안내 메시지 UX 원칙 (2026-04-05 확정)

**대상 사용자: 챗봇을 처음 사용하는 일반인 기준**

- 막연한 안내 금지: "다시 선택해주세요", "다른 유형을 선택해주세요" 등 행동 지침 없는 문구 금지
- 반드시 구체적 행동 지침 포함: 뭐라고 입력해야 하는지 명시
- 거래 불가 안내 메시지 형식:
  ```
  [매물명]은 [매매/임대차] 전용 매물이에요.
  [임대차/매매] 거래로는 진행이 어렵답니다.

  다른 매물을 고르시려면 "이전"을 입력해주시면 돼요.
  ```
