# Phase 2 — 최소 FastAPI 로컬 환경 구축 + 대화 흐름 검증

> 참조: `.claude/plans/dev-direction.md`
> 목표: 최소 FastAPI 서버를 로컬에서 구축하고 완성프롬프트.md 기반 전체 흐름을 검증한다.
> Phase 3(협의 항목 결정)은 이 환경에서 계속 테스트한다.

---

## 작업 항목

### 1. 환경 준비

- ✅ Python 3.11+ 확인 (3.13.9)
- ✅ 의존성 설치
  - fastapi, uvicorn
  - google-generativeai (Gemini) 또는 anthropic (Claude Haiku)
  - google-api-python-client (Google Sheets)
  - python-dotenv
- ✅ .env 파일 구성
  - GEMINI_API_KEY 또는 ANTHROPIC_API_KEY
  - AI_MODEL (환경변수로 모델 전환)
  - GOOGLE_SHEETS_ID / GOOGLE_CREDENTIALS
- ✅ google-generativeai → google-genai 교체 (deprecated 대응)

### 2. 최소 FastAPI 서버 구현

- ✅ 기본 chat endpoint 구현 (`POST /chat`)
- ✅ 인메모리 세션 관리 (Python dict)
  - {user_id: {step, collected_data, conversation_history}}
- ✅ 완성프롬프트.md → 시스템 프롬프트 로드
- ✅ HTML 채팅 페이지 추가 (`GET /` — 브라우저 수동 테스트용)
- ✅ 서버 기동 스크립트 (`run_server.py`) — 포트 8000 고정, sys.path/chdir 명시
- ✅ ngrok 외부 접속 환경 구축 (테스트용, 외부 테스터 공유 보류)
- 🔲 카카오 5초 타임아웃 대응 확인
- 🔲 프롬프트 캐싱 적용 (Gemini 또는 Claude)

### 3. 매물특징 파일 분기 구조 구현

- ✅ `.Source-Files/매물특징/` 디렉토리 생성
- ✅ 매물특징.md → 10개 파일로 분리
  - `1-아파트-주상복합-오피스텔-도시형-다세대.md`
  - `2-다가구-상가주택-단독주택.md`
  - `3-원룸-투룸-쓰리룸.md`
  - `4-수익성건물.md`
  - `5-구분상가-사무실-지식산업센터-오피스텔.md`
  - `6-공장-창고.md`
  - `7-토지.md`
  - `8-분양권.md`
  - `9-재개발-재건축-입주권.md`
  - `10-기타.md`
- ✅ 사용자 매물 선택 시 해당 번호 파일만 로드 → 시스템 프롬프트에 동적 주입

### 4. 대화 흐름 검증

> ⚠️ 배포 우선 전략 채택 (2026-04-04) — 개발자 단독 검증 대신 Phase 4 실서비스 배포 후 실사용 테스터 피드백으로 대체. 섹션 4~7은 Phase 4 이후 실사용 검증으로 갈음한다.

- 🔲 1단계 — 거래당사자 선택 동작 확인
- 🔲 2단계-A (매도인·임대인) — 주소 입력 흐름
- 🔲 2단계-B (매수인·임차인) — 매물종류 다중선택
- 🔲 3단계 — 매물종류 선택 (매도인·임대인)
- 🔲 4단계 — 기본정보 수집 (매물특징 파일 분기 동작 확인)
- 🔲 5단계 — 추가정보 수집 (금액·시기 표현 거래당사자별 확인)
- 🔲 6단계 — 특이사항
- 🔲 최종 단계 — 수집 정보 확인 및 접수

### 5. 공통 동작 검증

- 🔲 `다시` → 1단계 초기화
- 🔲 `이전` → 이전 단계 복귀
- 🔲 범위 밖 번호 입력 → 오류 안내
- 🔲 거래당사자별 표현 자동 조정 확인
  - 매도인: 양도가능시기 / 임대인: 입주가능시기
  - 매수인: 양수희망시기 / 임차인: 입주희망시기

### 6. 4가지 거래당사자 시나리오 테스트

- 🔲 매도인 → 주소 → 매물종류 → 기본정보 → 추가정보 → 확인 → 접수
- 🔲 매수인 → 매물종류 → 기본정보 → 추가정보 → 확인 → 접수
- 🔲 임대인 → 주소 → 매물종류 → 기본정보 → 추가정보 → 확인 → 접수
- 🔲 임차인 → 매물종류 → 기본정보 → 추가정보 → 확인 → 접수
- 🔲 복수 매물 선택 시 순차 진행 확인

### 7. 발견된 이슈 반영

- 🔲 테스트 결과 이슈 목록 작성
- 🔲 완성프롬프트.md 수정 반영
- 🔲 수정 후 재테스트
