# Phase 3 — 협의 항목 결정 및 프롬프트 완성

> 참조: `.claude/plans/dev-direction.md`
> 목표: 미결 항목을 모두 확정하고 완성프롬프트.md를 최종 완성한다.
> 테스트 환경: Phase 2에서 구축한 FastAPI 로컬 서버

---

## 작업 항목

### 미결 항목 결정 (협의 필요)

- 🔲 연락처 수집 여부 결정
  - 수집한다면: 어느 단계에서, 필수/선택 여부
  - 수집하지 않는다면: 접수 완료 후 중개사가 카카오 채널로 역연락
- 🔲 접수 완료 후 안내 방식 결정
  - 예: "접수가 완료되었습니다. 담당자가 확인 후 연락드리겠습니다."
  - 예상 연락 시간 안내 포함 여부
- 🔲 매물유형 거래당사자별 차별화 확정
  - 임대인: 분양권·재개발/재건축 제외 여부
  - 임차인: 분양권·재개발/재건축·토지 제외 여부
  - 일반건축물(임차인), 토지(임대인) 포함 여부

### 매물특징.md 보완

- 🔲 사용자 보유 자료와 대조
- 🔲 [확인 필요] 항목 확정 또는 제거
- 🔲 매물특징.md v0.3 업데이트

### 5단계 이후 프롬프트 구현 (10개 분류 기준)

- 🔲 1번 — 아파트·주상복합·오피스텔(주거용)·도시형·다세대 질문 흐름
- 🔲 2번 — 다가구·상가주택·단독주택 질문 흐름 (건물 전체 기준)
- 🔲 3번 — 원룸·투룸·쓰리룸 질문 흐름 (임대임차 전용)
- 🔲 4번 — 수익성건물 질문 흐름
- 🔲 5번 — 구분상가·사무실·지식산업센터·오피스텔(업무용) 질문 흐름
- 🔲 6번 — 공장·창고 질문 흐름
- 🔲 7번 — 토지 질문 흐름
- 🔲 8번 — 분양권 질문 흐름
- 🔲 9번 — 재개발·재건축·입주권 질문 흐름
- 🔲 10번 — 기타 (자유 입력 유도)
- 🔲 연락처 수집 흐름 (결정된 경우)

### 최종 단계 완성

- 🔲 수집 정보 확인 화면 완성 (5단계 이후 항목 포함)
- 🔲 접수 완료 안내 문구 확정 및 반영
- 🔲 수정 요청 흐름 (항목 선택 → 해당 단계 복귀)

### 전체 테스트

- 🔲 매도인 전체 흐름 (1단계 → 최종 확인 → 접수 완료)
- 🔲 매수인 전체 흐름
- 🔲 임대인 전체 흐름
- 🔲 임차인 전체 흐름
- 🔲 복수 매물 유형 선택 시 순차 진행 확인
- 🔲 "수정하겠습니다" 선택 → 해당 항목 재입력 흐름
