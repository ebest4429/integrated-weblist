## 확정된 설계 결정사항

### 결정 1 — CLI 필드 구조

```json
"cli": {
  "install_win": "winget install PostgreSQL.PostgreSQL",
  "install_mac": "brew install postgresql",
  "commands": [
    { "label": "연결", "code": "psql -U postgres -h HOST -d DBNAME" },
    { "label": "쿼리 실행", "code": "psql -c \"SELECT version();\"" }
  ],
  "info_url": "https://www.postgresql.org/docs/current/app-psql.html"
}
```

| 서브필드 | 설명 | 필수 여부 |
|---------|------|---------|
| `install_win` | Windows 설치 명령 (winget 또는 공식 인스톨러 링크) | 하나 이상 필수 |
| `install_mac` | macOS 설치 명령 (brew) | 선택 |
| `commands` | 주요 명령어 배열 — label + code 쌍 | 하나 이상 필수 |
| `info_url` | 공식 CLI 문서 링크 | 선택 |

> **규칙**: install_win 또는 install_mac 중 하나 이상 + commands 하나 이상 필수.
> 조건 미충족 시 cli 필드 자체 생략.

### 결정 2 — CLI 포함 기준

1. 공식 CLI 우선 포함
2. 서드파티도 포함하되 **보편적 활용도 기준**:
   - 대상이 소수 → 전부 포함
   - 대상이 다수 → 안정성·GitHub Star·다운로드 빈도 기준으로 선별, 빈도 낮은 것 제외
3. **Windows 기준 필수**, macOS는 있으면 추가

### 결정 3 — 작업 범위

현재 서비스(338개) 대부분은 CLI 미지원 예상.
두 가지 방향으로 분리:
- **기존 서비스 필드 추가**: CLI 있는 서비스에 cli 필드 추가
- **신규 서비스 추가**: 현재 목록에 없는 CLI 특화 서비스 일부 추가 (소수 예상)

> 전수 조사 후 해당 서비스 수에 따라 단일 조사 파일 vs 카테고리별 파일 결정.
> MCP보다 적을 경우 단일 파일로 처리.

### 결정 4 — 가상 필터뷰

```
기존 카테고리: JSON 파일 기반 (변경 없음)
신규 가상 뷰:
  - "CLI 도구"  → 전체 JSON에서 cli 필드 있는 서비스 수집
  - "MCP 연결"  → 전체 JSON에서 mcp 필드 있는 서비스 수집 (데이터 이미 완성)
```

- 기존 카테고리 구조 변경 없음
- 사이드바 최하단 별도 그룹으로 추가
- 카드 표시 방식: 기존과 동일 (모든 정보 표시)
- 다중 태그 시스템, 전면 구조 재설계: **제외**

### 결정 5 — UI 변경 범위 (최소)

| 파일 | 변경 내용 |
|------|---------|
| `App.jsx` | 가상 카테고리 처리 로직 추가 (~30줄) |
| `Sidebar.jsx` | 가상 뷰 그룹 + 항목 2개 추가 |
| `ItemDetail.jsx` | CLI 섹션 추가 (MCP 섹션 패턴 재사용) |

> CopyButton 컴포넌트는 Phase 4에서 이미 구현됨 — 재사용.
