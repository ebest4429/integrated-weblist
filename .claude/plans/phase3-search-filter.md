# Phase 3 — 검색 고도화 + 태그 필터

> Phase 2 완료(338개 detail 작성) 후 진입.
> 마스터플랜 Phase 3 항목 기반으로 작성.

---

## 목표

검색 기능을 고도화하고 뱃지·그룹 기반 태그 필터를 추가하여
338개 서비스를 더 빠르고 정밀하게 탐색할 수 있게 한다.

---

## 배경 결정사항

| 항목 | 결정 내용 |
|------|---------|
| 검색 고도화 범위 | 기존 name+desc 실시간 필터링에 뱃지·그룹 조건 추가. 취지: 사용자가 "무료 AI 도구"처럼 조건을 조합해 탐색하는 패턴 대응 |
| 태그 필터 방식 | 뱃지 필터(무료/유료/무료+유료/오픈소스/실험적) + 그룹 필터(AI/개발/마케팅/크리에이티브/구글) 체크박스. 취지: 카테고리 클릭 외에 횡단 검색 가능 |
| URL 상태 유지 | 검색어·필터 상태를 URL 파라미터로 저장. 취지: 북마크·공유 가능하게 |
| 검색 결과 UX | 매칭 텍스트 하이라이트. 취지: 어느 필드에서 매칭됐는지 시각적 피드백 |

## 작업 항목

### P3-1. 뱃지 필터 UI

- ✅ 뱃지 필터 컴포넌트 (FilterBar.jsx) 구현
  - 무료 / 유료 / 무료+유료 / 오픈소스 / 실험적 / Deprecated 토글 버튼
  - 전체 선택/해제 버튼
- ✅ 선택된 뱃지로 아이템 필터링 로직 연동 (App.jsx)
- ✅ 필터 상태 LocalStorage 저장

### P3-2. 그룹 필터

- ✅ 그룹 필터 토글 (AI관련 / 개발관련 / 마케팅·비즈니스 / 크리에이티브 / 구글생태계)
- ✅ 사이드바 그룹 선택과 연동 (사이드바 클릭 = 그룹 필터 자동 설정)

### P3-3. 검색 고도화

- ✅ 검색 대상 확장: name + nameKo + desc + detail.intro + detail.features 포함
- ✅ 검색 결과 하이라이트 — 매칭 텍스트 강조 표시
- ✅ 검색 결과 없음 안내 메시지

### P3-4. URL 상태 유지

- ✅ 검색어·뱃지 필터·그룹 필터 상태 URL 파라미터 반영 (`?q=...&badge=...&group=...`)
- ✅ URL 직접 접근 시 상태 복원

### P3-5. 검색·필터 결과 통계

- ✅ 현재 표시 중인 아이템 수 표시 ("전체 338개 중 N개")
- ✅ 필터 활성 시 리셋 버튼 표시

### P3-6. 빌드 최적화 (선택)

- ✅ JSON 데이터 동적 import (code splitting) — 청크 500kB 초과 경고 해소
  > 취지: 현재 빌드 시 520kB 단일 청크 경고. 카테고리별 lazy load로 초기 로드 개선.
  > 선택 항목: UX 영향 측정 후 진행 여부 결정.
