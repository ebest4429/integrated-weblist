알고리즘적 철학은 코드를 통해 표현되는 컴퓨테이셔널 미학 운동입니다. 출력 결과물은 .md 파일(철학), .html 파일(인터랙티브 뷰어), .js 파일(제너러티브 알고리즘)입니다.

작업은 두 단계로 이루어집니다:
1. 알고리즘 철학 창작 (.md 파일)
2. p5.js 제너러티브 아트로 표현 (.html + .js 파일)

먼저 다음 과정을 수행하세요:

## 알고리즘 철학 창작

시작점으로, 다음을 통해 해석될 알고리즘 철학(정적 이미지나 템플릿이 아닌)을 창작하세요:
- 컴퓨테이셔널 프로세스, 창발적 행동, 수학적 아름다움
- 시드 기반 무작위성, 노이즈 필드, 유기적 시스템
- 파티클, 플로우, 필드, 힘
- 파라메트릭 변화와 통제된 혼돈

### 핵심 이해

- 받는 것: 사용자의 미묘한 입력이나 고려할 지침 — 기반으로 삼되, 창작의 자유를 제한해서는 안 됩니다.
- 창작하는 것: 알고리즘 철학/제너러티브 미학 운동.
- 다음 단계: 동일한 버전이 철학을 받아 코드로 표현 — 90%는 알고리즘 생성, 10%는 핵심 파라미터인 p5.js 스케치 제작.

다음 접근 방식을 고려하세요:
- 제너러티브 아트 운동의 선언문 작성
- 다음 단계에서 이를 생동감 있게 만드는 알고리즘 작성

철학은 반드시 강조해야 합니다: 알고리즘적 표현. 창발적 행동. 컴퓨테이셔널 아름다움. 시드 기반 변주.

### 알고리즘 철학 생성 방법

**운동 이름 짓기** (1-2 단어): "유기적 난류" / "양자 조화" / "창발적 고요"

**철학 서술** (4-6 단락 — 간결하지만 완전하게):

이 철학의 알고리즘적 본질을 포착하기 위해, 다음을 통해 어떻게 표현되는지 표현하세요:
- 컴퓨테이셔널 프로세스와 수학적 관계?
- 노이즈 함수와 무작위성 패턴?
- 파티클 행동과 필드 다이나믹스?
- 시간적 진화와 시스템 상태?
- 파라메트릭 변화와 창발적 복잡성?

**필수 지침:**
- **중복 피하기**: 각 알고리즘적 측면은 한 번만 언급하세요. 노이즈 이론, 파티클 다이나믹스, 수학 원리에 대한 개념을 새로운 깊이를 추가하지 않는 한 반복하지 마세요.
- **장인 정신 반복 강조**: 철학은 최종 알고리즘이 수없이 많은 시간을 들여 개발되고, 세심하게 다듬어졌으며, 절대적 최고 수준의 전문가에게서 나온 것처럼 보여야 한다고 여러 번 강조해야 합니다. 이 프레이밍은 필수적입니다 — "세심하게 제작된 알고리즘", "깊은 컴퓨테이셔널 전문성의 산물", "고통스러운 최적화", "마스터 수준의 구현" 같은 표현을 반복하세요.
- **창작 공간 남기기**: 알고리즘적 방향에 대해 구체적이되, 다음 Claude가 극히 높은 수준의 장인 정신으로 해석적 구현 선택을 할 수 있을 만큼 간결하게.

철학은 다음 버전이 정적 이미지가 아닌 알고리즘으로 아이디어를 표현하도록 안내해야 합니다. 아름다움은 프로세스에 있지, 최종 프레임에 있지 않습니다.

### 철학 예시

**"유기적 난류"**
철학: 자연 법칙에 의해 제약된 혼돈, 무질서에서 질서가 창발함.
알고리즘적 표현: 레이어드 Perlin 노이즈로 구동되는 플로우 필드. 수천 개의 파티클이 벡터 힘을 따라 이동하며 흔적이 유기적 밀도 맵으로 축적됨. 여러 노이즈 옥타브가 난류 영역과 고요한 구역을 만들어냄. 색상은 속도와 밀도에서 창발 — 빠른 파티클은 밝게 타오르고, 느린 것들은 그림자로 사라짐. 알고리즘은 균형점까지 실행 — 컴퓨테이셔널 미학의 마스터가 수없이 많은 반복을 통해 세심하게 조율한 모든 파라미터의 균형.

**"양자 조화"**
철학: 파동 같은 간섭 패턴을 보이는 이산적 개체들.
알고리즘적 표현: 그리드에 초기화된 파티클, 각각 사인파를 통해 진화하는 위상값을 가짐. 파티클들이 가까워지면 위상이 간섭 — 건설적 간섭은 밝은 노드, 파괴적 간섭은 공허를 만듦. 단순 조화 운동이 복잡한 창발적 만다라를 생성. 모든 비율이 공명하는 아름다움을 만들어내도록 신중하게 선택된 고통스러운 주파수 조율의 결과.

**"재귀적 속삭임"**
철학: 규모를 넘나드는 자기 유사성, 유한한 공간 속의 무한한 깊이.
알고리즘적 표현: 재귀적으로 세분화되는 분기 구조. 각 가지는 황금 비율에 의해 제약되지만 약간씩 무작위화됨. L-시스템이나 재귀적 분할이 수학적이면서도 유기적인 느낌의 나무 같은 형태를 생성. 미묘한 노이즈 교란이 완벽한 대칭을 깨뜨림. 선의 두께는 재귀 수준마다 감소. 모든 분기 각도가 깊은 수학적 탐구의 산물.

**"필드 다이나믹스"**
철학: 물질에 미치는 영향을 통해 가시화된 보이지 않는 힘.
알고리즘적 표현: 수학적 함수나 노이즈에서 구성된 벡터 필드. 경계에서 태어나 필드 선을 따라 흐르다 균형점이나 경계에 도달하면 죽는 파티클. 여러 필드가 파티클을 끌어당기거나, 밀어내거나, 회전시킬 수 있음. 시각화는 오직 흔적만 보여줌 — 보이지 않는 힘의 유령 같은 증거. 힘의 균형을 통해 세심하게 안무된 컴퓨테이셔널 춤.

**"확률론적 결정화"**
철학: 무작위 프로세스가 질서 있는 구조로 결정화됨.
알고리즘적 표현: 무작위화된 원 패킹 또는 Voronoi 테셀레이션. 무작위 점으로 시작해 이완 알고리즘을 통해 진화. 셀들이 균형점까지 밀려남. 셀 크기, 이웃 수, 중심으로부터의 거리에 따라 색상 결정. 창발하는 유기적 타일링은 무작위적이면서도 필연적으로 느껴짐. 모든 시드가 고유한 결정체 아름다움을 만들어냄 — 마스터 수준의 제너러티브 알고리즘의 표식.

*이것들은 압축된 예시입니다. 실제 알고리즘 철학은 4-6개의 실질적인 단락이어야 합니다.*

### 필수 원칙
- **알고리즘 철학**: 코드를 통해 표현될 컴퓨테이셔널 세계관 창조
- **결과보다 프로세스**: 아름다움은 알고리즘의 실행에서 창발함을 항상 강조 — 매 실행이 고유함
- **파라메트릭 표현**: 아이디어는 수학적 관계, 힘, 행동을 통해 소통 — 정적 구성이 아닌
- **예술적 자유**: 다음 Claude가 철학을 알고리즘적으로 해석 — 창의적 구현 공간 제공
- **순수 제너러티브 아트**: 무작위성이 있는 정적 이미지가 아닌 살아있는 알고리즘을 만드는 것
- **전문가적 장인 정신**: 최종 알고리즘이 세심하게 제작되고, 수없이 많은 반복을 통해 다듬어졌으며, 컴퓨테이셔널 미학의 절대적 최고 수준 전문가의 깊은 전문성의 산물처럼 느껴져야 함을 반복 강조

**알고리즘 철학은 4-6 단락이어야 합니다.** 의도한 비전을 하나로 모으는 시적인 컴퓨테이셔널 철학으로 채우세요. 같은 점을 반복하지 마세요. 이 알고리즘 철학을 .md 파일로 출력하세요.

## 개념적 씨앗 도출

**중요 단계**: 알고리즘을 구현하기 전에, 원래 요청에서 미묘한 개념적 실마리를 식별하세요.

**핵심 원칙**:
개념은 알고리즘 자체 내에 내재된 **미묘하고 틈새적인 참조** — 항상 문자 그대로가 아니라, 항상 세련됩니다. 그 주제에 친숙한 사람은 직관적으로 느낄 수 있고, 다른 이들은 그저 훌륭한 제너러티브 구성을 경험합니다. 알고리즘 철학은 컴퓨테이셔널 언어를 제공합니다. 도출된 개념은 영혼을 제공 — 파라미터, 행동, 창발 패턴에 보이지 않게 짜여진 조용한 개념적 DNA.

이것은 **매우 중요합니다**: 참조는 작품의 깊이를 스스로 알리지 않고 강화할 만큼 정제되어야 합니다. 알고리즘적 조화를 통해 다른 노래를 인용하는 재즈 음악가처럼 생각하세요 — 아는 사람만 알아채지만, 모두가 제너러티브 아름다움을 감상합니다.

## P5.JS 구현

철학과 개념적 프레임워크가 확립되면, 코드를 통해 표현하세요. 진행 전에 생각을 정리할 시간을 가지세요. 창작된 알고리즘 철학과 아래 지침만 사용하세요.

### ⚠️ 0단계: 먼저 템플릿 읽기 ⚠️

**중요: HTML을 작성하기 전에:**

1. Read 도구로 `templates/viewer.html` **읽기**
2. 정확한 구조, 스타일링, Anthropic 브랜딩 **학습**
3. 그 파일을 영감이 아닌 **문자 그대로의 시작점**으로 사용
4. **고정된 섹션을 파일의 주석에 표시된 대로 정확히 유지** (헤더, 사이드바 구조, Anthropic 색상/폰트, 시드 컨트롤, 액션 버튼)
5. 파일의 주석에 표시된 **변수 섹션만 교체** (알고리즘, 파라미터, 파라미터용 UI 컨트롤)

**피해야 할 것:**
- ❌ 처음부터 HTML 만들기
- ❌ 커스텀 스타일링이나 색상 체계 발명
- ❌ 시스템 폰트나 다크 테마 사용
- ❌ 사이드바 구조 변경

**따라야 할 것:**
- ✅ 템플릿의 정확한 HTML 구조 복사
- ✅ Anthropic 브랜딩 유지 (Poppins/Lora 폰트, 밝은 색상, 그래디언트 배경)
- ✅ 사이드바 레이아웃 유지 (시드 → 파라미터 → 색상? → 액션)
- ✅ p5.js 알고리즘과 파라미터 컨트롤만 교체

템플릿이 기반입니다. 재구축하지 말고 그 위에 구축하세요.

갤러리 수준의 컴퓨테이셔널 아트를 만들기 위해, 알고리즘 철학을 기반으로 사용하세요.

### 기술 요구사항

**시드 기반 무작위성 (Art Blocks 패턴)**:
```javascript
// 재현성을 위해 항상 시드 사용
let seed = 12345; // 또는 사용자 입력의 해시
randomSeed(seed);
noiseSeed(seed);
```

**파라미터 구조 — 철학을 따르세요**:

철학에서 자연스럽게 창발하는 파라미터를 설정하기 위해, "이 시스템의 어떤 속성을 조정할 수 있는가?"를 고려하세요.

```javascript
let params = {
  seed: 12345,  // 재현성을 위해 항상 시드 포함
  // 색상
  // 알고리즘을 제어하는 파라미터 추가:
  // - 수량 (얼마나 많이?)
  // - 스케일 (얼마나 크게? 얼마나 빠르게?)
  // - 확률 (얼마나 가능성 있게?)
  // - 비율 (어떤 비례?)
  // - 각도 (어떤 방향?)
  // - 임계값 (언제 행동이 변하는가?)
};
```

**파라미터를 효과적으로 설계하려면, "패턴 유형"으로 생각하지 말고 시스템이 조율 가능해야 하는 속성에 집중하세요.**

**핵심 알고리즘 — 철학을 표현하세요**:

**중요**: 알고리즘 철학이 무엇을 만들지 지시해야 합니다.

철학을 코드로 표현하기 위해, "어떤 패턴을 사용해야 할까?"가 아닌 "이 철학을 어떻게 코드로 표현할까?"를 생각하세요.

철학이 **유기적 창발**에 관한 것이라면, 다음 사용 고려:
- 시간이 지남에 따라 축적되거나 성장하는 요소
- 자연 법칙에 의해 제약된 무작위 프로세스
- 피드백 루프와 상호작용

철학이 **수학적 아름다움**에 관한 것이라면, 다음 사용 고려:
- 기하학적 관계와 비율
- 삼각함수와 조화
- 예상치 못한 패턴을 만드는 정밀한 계산

철학이 **통제된 혼돈**에 관한 것이라면, 다음 사용 고려:
- 엄격한 경계 내의 무작위 변화
- 분기와 위상 전환
- 무질서에서 창발하는 질서

**알고리즘은 선택 메뉴가 아닌 철학에서 흘러나옵니다.**

구현을 안내하기 위해, 개념적 본질이 창의적이고 독창적인 선택을 알리도록 하세요. 이 특정 요청의 비전을 표현하는 것을 구축하세요.

**캔버스 설정**: 표준 p5.js 구조:
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 시스템 초기화
}

function draw() {
  // 제너러티브 알고리즘
  // 정적(noLoop) 또는 애니메이션 가능
}
```

### 장인 정신 요구사항

**중요**: 마스터 제너러티브 아티스트가 수없이 많은 반복을 통해 창출한 것처럼 느껴지는 알고리즘을 만드세요. 모든 파라미터를 세심하게 조율하세요. 모든 패턴이 의도를 가지고 창발하도록 하세요. 이것은 무작위 노이즈가 아닙니다 — 깊은 전문성을 통해 정제된 통제된 혼돈입니다.

- **균형**: 시각적 노이즈 없는 복잡성, 경직성 없는 질서
- **색상 조화**: 무작위 RGB 값이 아닌 사려 깊은 팔레트
- **구성**: 무작위성 속에서도 시각적 위계와 흐름 유지
- **성능**: 부드러운 실행, 애니메이션인 경우 실시간 최적화
- **재현성**: 같은 시드는 항상 동일한 출력 생성

### 출력 형식

출력:
1. **알고리즘 철학** — 제너러티브 미학을 설명하는 마크다운 또는 텍스트
2. **단일 HTML 아티팩트** — `templates/viewer.html`을 기반으로 구축된 자체 완결형 인터랙티브 제너러티브 아트 (0단계와 다음 섹션 참조)

HTML 아티팩트는 모든 것을 포함합니다: p5.js (CDN에서), 알고리즘, 파라미터 컨트롤, UI — 모두 claude.ai 아티팩트나 어떤 브라우저에서도 즉시 작동하는 하나의 파일. 처음부터 시작하지 말고 템플릿 파일에서 시작하세요.

## 인터랙티브 아티팩트 창작

**알림: `templates/viewer.html`은 이미 읽었어야 합니다 (0단계 참조). 그 파일을 시작점으로 사용하세요.**

제너러티브 아트의 탐색을 허용하기 위해, 단일 자체 완결형 HTML 아티팩트를 만드세요. 이 아티팩트가 claude.ai나 어떤 브라우저에서도 즉시 작동하도록 하세요 — 설정 불필요. 모든 것을 인라인으로 임베드하세요.

### 중요: 고정 vs 변수 항목

`templates/viewer.html` 파일이 기반입니다. 필요한 정확한 구조와 스타일링을 포함합니다.

**고정 (항상 표시된 대로 정확히 포함):**
- 레이아웃 구조 (헤더, 사이드바, 메인 캔버스 영역)
- Anthropic 브랜딩 (UI 색상, 폰트, 그래디언트)
- 사이드바의 시드 섹션:
  - 시드 표시
  - 이전/다음 버튼
  - 무작위 버튼
  - 시드로 이동 입력 + 이동 버튼
- 사이드바의 액션 섹션:
  - 재생성 버튼
  - 초기화 버튼

**변수 (각 아트워크마다 커스터마이즈):**
- 전체 p5.js 알고리즘 (setup/draw/클래스)
- 파라미터 객체 (아트에 필요한 것 정의)
- 사이드바의 파라미터 섹션:
  - 파라미터 컨트롤 수
  - 파라미터 이름
  - 슬라이더의 최소/최대/단계 값
  - 컨트롤 유형 (슬라이더, 입력 등)
- 색상 섹션 (선택적):
  - 일부 아트는 컬러 피커 필요
  - 일부 아트는 고정 색상 사용
  - 일부 아트는 단색 (색상 컨트롤 불필요)
  - 아트의 필요에 따라 결정

**모든 아트워크는 고유한 파라미터와 알고리즘을 가져야 합니다!** 고정 부분은 일관된 UX 제공 — 그 외 모든 것이 고유한 비전을 표현합니다.

### 필수 기능

**1. 파라미터 컨트롤**
- 숫자 파라미터용 슬라이더 (파티클 수, 노이즈 스케일, 속도 등)
- 팔레트 색상용 컬러 피커
- 파라미터 변경 시 실시간 업데이트
- 기본값으로 되돌리는 초기화 버튼

**2. 시드 탐색**
- 현재 시드 번호 표시
- 시드를 순환하는 "이전"과 "다음" 버튼
- 무작위 시드를 위한 "무작위" 버튼
- 특정 시드로 이동하는 입력 필드
- 요청 시 100가지 변주 생성 (시드 1-100)

**3. 단일 아티팩트 구조**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js CDN에서 — 항상 사용 가능 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 모든 스타일 인라인 — 깔끔하고 미니멀 */
    /* 캔버스 위, 컨트롤 아래 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 모든 파라미터 컨트롤 -->
  </div>
  <script>
    // 모든 p5.js 코드 인라인
    // 파라미터 객체, 클래스, 함수
    // setup()과 draw()
    // UI 핸들러
    // 모두 자체 완결형
  </script>
</body>
</html>
```

**중요**: 단일 아티팩트입니다. 외부 파일 없음, 임포트 없음 (p5.js CDN 제외). 모든 것이 인라인.

**4. 구현 세부사항 — 사이드바 구축**

사이드바 구조:

**1. 시드 (고정)** — 항상 표시된 대로 정확히 포함:
- 시드 표시
- 이전/다음/무작위/이동 버튼

**2. 파라미터 (변수)** — 아트를 위한 컨트롤 생성:
```html
<div class="control-group">
    <label>파라미터 이름</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
파라미터 수만큼 control-group div를 추가하세요.

**3. 색상 (선택적/변수)** — 아트에 조정 가능한 색상이 필요한 경우 포함:
- 사용자가 팔레트를 제어해야 하는 경우 컬러 피커 추가
- 아트가 고정 색상을 사용하는 경우 이 섹션 생략
- 아트가 단색인 경우 생략

**4. 액션 (고정)** — 항상 표시된 대로 정확히 포함:
- 재생성 버튼
- 초기화 버튼
- PNG 다운로드 버튼

**요구사항**:
- 시드 컨트롤이 작동해야 함 (이전/다음/무작위/이동/표시)
- 모든 파라미터에 UI 컨트롤이 있어야 함
- 재생성, 초기화, 다운로드 버튼이 작동해야 함
- Anthropic 브랜딩 유지 (UI 스타일링, 아트 색상 아님)

### 아티팩트 사용

HTML 아티팩트는 즉시 작동합니다:
1. **claude.ai에서**: 인터랙티브 아티팩트로 표시 — 즉시 실행
2. **파일로**: 저장하고 어떤 브라우저에서도 열기 — 서버 불필요
3. **공유**: HTML 파일 전송 — 완전 자체 완결형

## 변주 및 탐색

아티팩트는 기본적으로 시드 탐색을 포함합니다 (이전/다음/무작위 버튼), 여러 파일을 만들지 않고 변주를 탐색할 수 있습니다. 특정 변주를 강조하고 싶다면:

- 시드 프리셋 포함 ("변주 1: 시드 42", "변주 2: 시드 127" 버튼 등)
- 여러 시드의 썸네일을 나란히 보여주는 "갤러리 모드" 추가
- 모두 같은 단일 아티팩트 내에서

이것은 같은 판화에서 시리즈를 만드는 것과 같습니다 — 알고리즘은 일관되지만, 각 시드는 그 가능성의 다른 면을 드러냅니다. 인터랙티브한 특성은 사용자가 시드 공간을 탐색하며 자신이 좋아하는 것을 발견하게 합니다.

**사용자 요청** → **알고리즘 철학** → **구현**

각 요청은 고유합니다. 프로세스는 다음을 포함합니다:

1. **사용자의 의도 해석** — 어떤 미학을 추구하는가?
2. **알고리즘 철학 창작** (4-6 단락) 컴퓨테이셔널 접근 방식 설명

4. **적절한 파라미터 설계** — 무엇이 조율 가능해야 하는가?
5. **매칭되는 UI 컨트롤 구축** — 해당 파라미터를 위한 슬라이더/입력

**상수**:
- Anthropic 브랜딩 (색상, 폰트, 레이아웃)
- 시드 탐색 (항상 존재)
- 자체 완결형 HTML 아티팩트

**모든 것이 변수**:
- 알고리즘 자체
- 파라미터
- UI 컨트롤
- 시각적 결과

최선의 결과를 위해, 창의성을 믿고 철학이 구현을 안내하도록 하세요.
