# 통계 분석 스킬

기술 통계, 추세 분석, 이상값 탐지, 가설 검정, 그리고 통계적 주장을 할 때 주의해야 할 상황에 대한 가이드를 제공합니다.

## 기술 통계 방법론

### 중심 경향성

데이터 특성에 맞는 중심 측도를 선택합니다:

| 상황 | 사용할 측도 | 이유 |
|---|---|---|
| 대칭 분포, 이상값 없음 | 평균(Mean) | 가장 효율적인 추정량 |
| 치우친 분포 | 중앙값(Median) | 이상값에 강건함 |
| 범주형 또는 순서형 데이터 | 최빈값(Mode) | 비수치형 데이터의 유일한 옵션 |
| 이상값이 있는 고도 왜곡 데이터 (예: 사용자당 매출) | 중앙값 + 평균 | 둘 다 보고; 차이가 왜도를 나타냄 |

**비즈니스 지표에는 항상 평균과 중앙값을 함께 보고합니다.** 두 값이 크게 차이난다면 데이터가 편향되어 있으며, 평균만으로는 오해를 줄 수 있습니다.

### 산포도와 변동성

- **표준편차**: 값들이 평균에서 얼마나 떨어져 있는지. 정규 분포 데이터에 사용.
- **사분위 범위(IQR)**: p25에서 p75까지의 거리. 이상값에 강건함. 치우친 데이터에 사용.
- **변동계수(CV)**: 표준편차 / 평균. 척도가 다른 지표 간 변동성 비교에 사용.
- **범위(Range)**: 최댓값 - 최솟값. 이상값에 민감하지만 데이터 범위를 빠르게 파악하는 데 유용.

### 비즈니스 맥락의 백분위수

평균만으로는 부족한 정보를 보완하기 위해 핵심 백분위수를 보고합니다:

```
p1:   하위 1% (바닥/최솟값에 가까운 값)
p5:   정상 범위 하단
p25:  1사분위수
p50:  중앙값 (일반 사용자)
p75:  3사분위수
p90:  상위 10% / 헤비 유저
p95:  정상 범위 상단
p99:  상위 1% / 극단적 사용자
```

**서술 예시**: "중앙값 세션 시간은 4.2분이지만, 상위 10% 사용자는 세션당 22분 이상을 사용하여 평균을 7.8분으로 끌어올리고 있습니다."

### 분포 설명

분석하는 모든 수치 분포의 특성을 서술합니다:

- **모양**: 정규, 우편향, 좌편향, 이봉형, 균등, 두꺼운 꼬리
- **중심**: 평균과 중앙값 (및 두 값의 차이)
- **산포**: 표준편차 또는 IQR
- **이상값**: 개수와 극단 정도
- **경계**: 자연적 하한(0)이나 상한(100%)이 있는지 여부

## 추세 분석 및 예측

### 추세 식별

**노이즈 평활화를 위한 이동 평균:**
```python
# 7일 이동 평균 (주간 계절성이 있는 일별 데이터에 적합)
df['ma_7d'] = df['metric'].rolling(window=7, min_periods=1).mean()

# 28일 이동 평균 (주간 및 월간 패턴 모두 평활화)
df['ma_28d'] = df['metric'].rolling(window=28, min_periods=1).mean()
```

**기간 비교:**
- 전주 대비(WoW): 지난주 같은 요일과 비교
- 전월 대비(MoM): 전월 같은 달과 비교
- 전년 대비(YoY): 계절성 비즈니스의 황금 기준
- 전년 동일일 비교: 특정 달력 날짜 비교

**성장률:**
```
단순 성장률: (현재 - 이전) / 이전
CAGR: (최종값 / 시작값) ^ (1 / 연수) - 1
로그 성장률: ln(현재 / 이전)  -- 변동성이 큰 시계열에 적합
```

### 계절성 탐지

주기적 패턴 확인:
1. 원시 시계열 플롯 — 시각적 검사 우선
2. 요일별 평균 계산: 뚜렷한 주간 패턴이 있는가?
3. 월별 평균 계산: 연간 주기가 있는가?
4. 기간 비교 시 항상 YoY 또는 동기간 비교를 사용하여 추세와 계절성을 혼동하지 않도록 주의

### 예측 (간단한 방법)

비즈니스 분석가(데이터 과학자가 아닌 경우)를 위한 간단한 방법:

- **나이브 예측**: 내일 = 오늘. 기준선으로 사용.
- **계절 나이브**: 내일 = 지난주/지난해 같은 날.
- **선형 추세**: 과거 데이터에 직선 피팅. 명확히 선형 추세가 있을 때만 사용.
- **이동 평균 예측**: 후행 평균을 예측값으로 사용.

**항상 불확실성을 전달합니다.** 단일 수치가 아닌 범위를 제공합니다:
- "3개월 추세 기준 다음 달 가입자는 10,000~12,000명으로 예상됩니다"
- "다음 달 정확히 11,234명이 가입할 것입니다"라고 하지 마세요

**데이터 과학자에게 에스컬레이션해야 할 때**: 비선형 추세, 복수의 계절성, 외부 요인(마케팅 지출, 공휴일), 또는 자원 배분에 예측 정확도가 중요한 경우.

## 이상값 및 이상 탐지

### 통계적 방법

**Z-점수 방법** (정규 분포 데이터용):
```python
z_scores = (df['value'] - df['value'].mean()) / df['value'].std()
outliers = df[abs(z_scores) > 3]  # 3 표준편차 초과
```

**IQR 방법** (비정규 분포에 강건):
```python
Q1 = df['value'].quantile(0.25)
Q3 = df['value'].quantile(0.75)
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR
outliers = df[(df['value'] < lower_bound) | (df['value'] > upper_bound)]
```

**백분위수 방법** (가장 간단):
```python
outliers = df[(df['value'] < df['value'].quantile(0.01)) |
              (df['value'] > df['value'].quantile(0.99))]
```

### 이상값 처리

이상값을 자동으로 제거하지 마세요. 대신:

1. **조사**: 데이터 오류인지, 실제 극단값인지, 다른 모집단인지 파악
2. **데이터 오류**: 수정하거나 제거 (예: 음수 나이, 1970년 타임스탬프)
3. **실제 극단값**: 유지하되 강건한 통계량 사용 (평균 대신 중앙값)
4. **다른 모집단**: 별도 분석을 위해 세그먼트 분리 (예: 엔터프라이즈 vs. SMB 고객)

**수행한 작업을 명시합니다**: "거래 금액 $50K 초과인 47건(0.3%)을 제외했으며, 이는 별도 분석되는 대량 엔터프라이즈 주문입니다."

### 시계열 이상 탐지

시계열에서 비정상적인 값 탐지 방법:

1. 기대값 계산 (이동 평균 또는 전년 동기)
2. 기대값과의 편차 계산
3. 임계값(보통 잔차의 2~3 표준편차) 초과 편차 표시
4. 점 이상(단일 비정상 값)과 변화점(지속적 변화)을 구분

## 가설 검정 기초

### 사용 시기

관측된 차이가 실제인지 우연에 의한 것인지 판단할 때 가설 검정을 사용합니다. 주요 시나리오:

- A/B 테스트 결과: B 변형이 실제로 A보다 좋은가?
- 전후 비교: 제품 변경이 실제로 지표를 바꿨는가?
- 세그먼트 비교: 엔터프라이즈 고객이 실제로 더 높은 리텐션을 가지는가?

### 검정 프레임워크

1. **귀무 가설(H0)**: 차이가 없다 (기본 가정)
2. **대립 가설(H1)**: 차이가 있다
3. **유의 수준(alpha) 선택**: 보통 0.05 (5% 위양성 확률)
4. **검정 통계량과 p값 계산**
5. **해석**: p < alpha이면 H0 기각 (실제 차이의 증거)

### 주요 검정 방법

| 시나리오 | 검정 방법 | 사용 시기 |
|---|---|---|
| 두 그룹 평균 비교 | 독립 t-검정 | 정규 데이터, 두 그룹 |
| 두 그룹 비율 비교 | 비율 z-검정 | 전환율, 이진 결과 |
| 쌍 측정값 비교 | 대응 t-검정 | 동일 개체의 전후 비교 |
| 3개 이상 그룹 평균 비교 | ANOVA | 복수 세그먼트 또는 변형 |
| 비정규 데이터, 두 그룹 | Mann-Whitney U 검정 | 치우친 지표, 순서형 데이터 |
| 범주 간 연관성 | 카이제곱 검정 | 두 범주형 변수 |

### 실질적 유의성 vs. 통계적 유의성

**통계적 유의성**은 차이가 우연에 의한 것일 가능성이 낮다는 의미입니다.

**실질적 유의성**은 차이가 비즈니스 결정에 영향을 줄 만큼 크다는 의미입니다.

차이가 통계적으로는 유의하지만 실질적으로는 의미 없을 수 있습니다 (대용량 샘플에서 흔함). 항상 다음을 보고합니다:
- **효과 크기**: 차이가 얼마나 큰가? (예: "B 변형은 전환율을 0.3%포인트 개선했습니다")
- **신뢰 구간**: 실제 효과의 그럴듯한 범위는?
- **비즈니스 영향**: 매출, 사용자 수 등 비즈니스 용어로 환산하면?

### 표본 크기 고려사항

- 작은 표본은 유의한 p값이 나와도 신뢰할 수 없는 결과를 낳습니다
- 비율의 경우 경험칙: 기본 신뢰성을 위해 그룹당 최소 30건의 이벤트 필요
- 작은 효과(예: 1% 전환율 변화)를 탐지하려면 그룹당 수천 건의 관측값이 필요할 수 있습니다
- 표본이 작으면 명시하세요: "그룹당 200건의 관측값만으로는 X% 미만의 효과를 탐지할 통계적 검정력이 부족합니다"

## 통계적 주장 시 주의사항

### 상관관계는 인과관계가 아닙니다

상관관계를 발견했을 때 반드시 고려해야 할 사항:
- **역인과관계**: A가 B를 유발하는 게 아니라 B가 A를 유발할 수도 있습니다
- **교란 변수**: C가 A와 B 모두를 유발할 수 있습니다
- **우연의 일치**: 충분히 많은 변수가 있으면 허위 상관은 불가피합니다

**말할 수 있는 것**: "기능 X를 사용하는 사용자는 리텐션이 30% 높습니다"
**더 많은 증거 없이 말할 수 없는 것**: "기능 X가 리텐션을 30% 높입니다"

### 다중 비교 문제

많은 가설을 검정하면 일부는 우연히 "유의하게" 나타납니다:
- p=0.05 수준에서 20개 지표를 검정하면 약 1개는 거짓 유의
- 차이를 보이는 세그먼트를 찾기 전에 여러 세그먼트를 살펴봤다면 명시하세요
- 본페로니 보정(alpha를 검정 수로 나눔)으로 다중 비교를 조정하거나 실행한 검정 수를 보고하세요

### 심슨의 역설

집계 데이터의 추세가 데이터를 세분화하면 역전될 수 있습니다:
- 결론이 주요 세그먼트 전체에서 유지되는지 항상 확인
- 예시: 전체 전환율은 올라갔지만 모든 세그먼트에서는 내려감 — 전환율이 높은 세그먼트로 믹스가 이동했기 때문

### 생존자 편향

데이터셋에 "살아남은" 개체만 분석할 수 있습니다:
- 활성 사용자 분석은 이탈한 사용자를 제외합니다
- 성공한 기업 분석은 실패한 기업을 제외합니다
- 항상 질문하세요: "이 데이터셋에서 누가 빠져 있으며, 그들을 포함하면 결론이 달라지는가?"

### 생태학적 오류

집계 추세가 개인에게 적용되지 않을 수 있습니다:
- "X가 높은 국가들이 Y도 높다"는 것이 "X가 높은 개인들이 Y도 높다"는 것을 의미하지 않습니다
- 그룹 수준의 발견을 개인 사례에 적용할 때 주의하세요

### 특정 숫자에 집착하지 않기

거짓 정밀도를 경계합니다:
- "다음 분기 이탈률은 4.73%가 될 것입니다"는 정당화될 수 없는 확실성을 암시합니다
- 범위를 선호하세요: "과거 패턴에 근거하면 이탈률은 4~6%로 예상됩니다"
- 적절히 반올림하세요: "약 5%"가 종종 "4.73%"보다 더 정직합니다
