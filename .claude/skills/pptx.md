## 디자인 아이디어

**지루한 슬라이드는 만들지 마세요.** 흰 배경에 단순한 불릿 포인트는 아무도 감동시키지 못합니다. 각 슬라이드에 이 목록에서 아이디어를 참고하세요.

### 시작 전

- **주제에 맞는 강렬한 색상 팔레트 선택**: 팔레트는 이 주제를 위해 설계된 느낌이어야 합니다. 완전히 다른 프레젠테이션에 색상을 그대로 바꿔도 "통한다"면, 충분히 구체적인 선택이 아닙니다.
- **균등 배분보다 지배색**: 하나의 색상이 시각적 비중(60-70%)을 지배하고, 1~2개의 보조색과 하나의 강조색을 사용하세요. 모든 색상에 동일한 비중을 주면 안 됩니다.
- **명암 대비**: 제목과 결론 슬라이드에는 어두운 배경, 내용 슬라이드에는 밝은 배경("샌드위치" 구조). 또는 고급스러운 느낌을 위해 전체를 어두운 배경으로 통일.
- **시각적 모티프 일관성**: 하나의 독특한 요소를 선택하고 반복 사용하세요 — 둥근 이미지 프레임, 색상 원 안의 아이콘, 굵은 한쪽 테두리. 모든 슬라이드에 일관되게 적용하세요.

### 색상 팔레트

주제에 맞는 색상을 선택하세요 — 무난한 파란색을 기본값으로 사용하지 마세요. 다음 팔레트를 참고하세요:

| 테마 | 주색 | 보조색 | 강조색 |
|-------|---------|-----------|--------|
| **미드나잇 이그제큐티브** | `1E2761` (네이비) | `CADCFC` (아이스 블루) | `FFFFFF` (흰색) |
| **포레스트 & 모스** | `2C5F2D` (숲 초록) | `97BC62` (이끼) | `F5F5F5` (크림) |
| **코랄 에너지** | `F96167` (코랄) | `F9E795` (골드) | `2F3C7E` (네이비) |
| **웜 테라코타** | `B85042` (테라코타) | `E7E8D1` (모래) | `A7BEAE` (세이지) |
| **오션 그라디언트** | `065A82` (딥 블루) | `1C7293` (틸) | `21295C` (미드나잇) |
| **차콜 미니멀** | `36454F` (차콜) | `F2F2F2` (오프화이트) | `212121` (검정) |
| **틸 트러스트** | `028090` (틸) | `00A896` (시폼) | `02C39A` (민트) |
| **베리 & 크림** | `6D2E46` (베리) | `A26769` (더스티 로즈) | `ECE2D0` (크림) |
| **세이지 캄** | `84B59F` (세이지) | `69A297` (유칼립투스) | `50808E` (슬레이트) |
| **체리 볼드** | `990011` (체리) | `FCF6F5` (오프화이트) | `2F3C7E` (네이비) |

### 슬라이드별 구성

**모든 슬라이드에는 시각적 요소가 필요합니다** — 이미지, 차트, 아이콘, 도형 중 하나. 텍스트만 있는 슬라이드는 기억에 남지 않습니다.

**레이아웃 옵션:**
- 2단 구성 (왼쪽 텍스트, 오른쪽 일러스트)
- 아이콘 + 텍스트 행 (색상 원 안의 아이콘, 굵은 헤더, 아래 설명)
- 2×2 또는 2×3 그리드 (한쪽에 이미지, 다른 쪽에 콘텐츠 블록 그리드)
- 하프 블리드 이미지 (전체 왼쪽 또는 오른쪽) + 콘텐츠 오버레이

**데이터 표시:**
- 대형 통계 콜아웃 (60-72pt 큰 숫자, 아래에 작은 레이블)
- 비교 컬럼 (이전/이후, 장단점, 나란히 옵션 비교)
- 타임라인 또는 프로세스 흐름 (번호 매긴 단계, 화살표)

**시각적 완성도:**
- 섹션 헤더 옆 작은 색상 원 안의 아이콘
- 주요 통계나 태그라인에 이탤릭체 강조 텍스트

### 타이포그래피

**흥미로운 폰트 조합을 선택하세요** — Arial을 기본값으로 쓰지 마세요. 개성 있는 헤더 폰트를 선택하고 깔끔한 본문 폰트와 페어링하세요.

| 헤더 폰트 | 본문 폰트 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 요소 | 크기 |
|---------|------|
| 슬라이드 제목 | 36-44pt 굵게 |
| 섹션 헤더 | 20-24pt 굵게 |
| 본문 텍스트 | 14-16pt |
| 캡션 | 10-12pt 흐리게 |

### 여백

- 최소 0.5" 마진
- 콘텐츠 블록 사이 0.3-0.5"
- 여유 공간을 남기세요 — 모든 공간을 채우지 않아도 됩니다

### 피해야 할 것 (흔한 실수)

- **같은 레이아웃 반복 금지** — 슬라이드마다 컬럼, 카드, 콜아웃을 다양하게 변형하세요
- **본문 텍스트 가운데 정렬 금지** — 단락과 목록은 왼쪽 정렬, 제목만 가운데 정렬
- **크기 대비 부족 금지** — 14-16pt 본문과 구분되려면 제목은 36pt 이상 필요
- **무난한 파란색 기본값 금지** — 해당 주제를 반영하는 색상 선택
- **무작위 여백 금지** — 0.3" 또는 0.5" 간격을 선택하고 일관되게 적용
- **한 슬라이드만 스타일 적용 금지** — 전체에 완전히 적용하거나 심플하게 유지
- **텍스트만 있는 슬라이드 금지** — 이미지, 아이콘, 차트, 시각 요소 추가; 단순 제목+불릿 금지
- **텍스트 박스 패딩 무시 금지** — 선이나 도형을 텍스트 가장자리에 정렬할 때 텍스트 박스에 `margin: 0`을 설정하거나 패딩을 고려해 도형 위치를 조정하세요
- **낮은 대비 요소 금지** — 아이콘과 텍스트 모두 배경과 강한 대비가 필요합니다; 밝은 배경에 밝은 텍스트나 어두운 배경에 어두운 텍스트 사용 금지
- **제목 아래 강조선 절대 사용 금지** — AI 생성 슬라이드의 특징; 여백이나 배경 색상을 대신 사용하세요

## QA (필수)

**문제가 있다고 가정하세요. 당신의 역할은 찾아내는 것입니다.**

첫 번째 렌더링이 올바를 경우는 거의 없습니다. QA를 확인 단계가 아닌 버그 사냥으로 접근하세요. 첫 번째 검사에서 아무 문제도 발견하지 못했다면, 충분히 꼼꼼하게 보지 않은 것입니다.

### 내용 QA

```bash
python -m markitdown output.pptx
```

누락된 내용, 오타, 잘못된 순서를 확인하세요.

**템플릿 사용 시 남은 플레이스홀더 텍스트 확인:**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

grep에 결과가 나오면 완료 선언 전에 수정하세요.

### 시각적 QA

**⚠️ 서브에이전트 사용** — 슬라이드가 2-3개뿐이어도. 코드를 계속 보다 보면 기대하는 것을 보게 되어 실제 있는 것을 놓칩니다. 서브에이전트는 신선한 시각을 가집니다.

슬라이드를 이미지로 변환하고 ([이미지로 변환](#이미지로-변환) 참조), 다음 프롬프트를 사용하세요:

```
이 슬라이드를 시각적으로 검사하세요. 문제가 있다고 가정하고 찾아내세요.

다음을 확인하세요:
- 겹치는 요소 (도형을 통과하는 텍스트, 단어를 통과하는 선, 쌓인 요소)
- 텍스트 오버플로우 또는 가장자리/박스 경계에서 잘린 텍스트
- 제목이 두 줄로 줄 바꿈되었는데 한 줄 텍스트용으로 배치된 장식선
- 위의 내용과 충돌하는 출처 인용 또는 푸터
- 너무 가까운 요소 (0.3" 미만 간격) 또는 거의 닿을 듯한 카드/섹션
- 불균등한 간격 (한 곳은 넓고 다른 곳은 좁음)
- 슬라이드 가장자리에서 불충분한 마진 (0.5" 미만)
- 컬럼 또는 유사 요소가 일관되게 정렬되지 않음
- 낮은 대비 텍스트 (예: 크림색 배경에 밝은 회색 텍스트)
- 낮은 대비 아이콘 (예: 대비 원 없이 어두운 배경에 어두운 아이콘)
- 텍스트 박스가 너무 좁아 과도한 줄 바꿈 발생
- 남은 플레이스홀더 내용

각 슬라이드에 대해 사소한 것도 포함하여 문제 또는 우려 사항을 나열하세요.

이 이미지들을 읽고 분석하세요 — ls slide-*.jpg 에서 정확한 파일명을 사용하세요 (패딩이 다를 수 있음: slide-1 vs slide-01):
1. <slide-N>.jpg — (예상: [간략한 설명])
2. <slide-N>.jpg — (예상: [간략한 설명])
...

사소한 것도 포함하여 발견된 모든 문제를 보고하세요.
```

### 검증 루프

1. 슬라이드 생성 → 이미지로 변환 → 검사
2. **발견된 문제 목록 작성** (없으면 더 꼼꼼하게 다시 확인)
3. 문제 수정
4. **영향받은 슬라이드 재검증** — 하나를 고치면 종종 다른 문제가 생깁니다
5. 전체 검사에서 새로운 문제가 없을 때까지 반복

**최소 하나의 수정-검증 사이클을 완료할 때까지 완료를 선언하지 마세요.**

## 이미지로 변환

시각적 검사를 위해 프레젠테이션을 개별 슬라이드 이미지로 변환합니다:

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
ls slide-*.jpg
```

**`ls`에서 출력된 경로를 사용하세요.** `pdftoppm`은 페이지 수에 따라 숫자에 0을 채웁니다: 10페이지 미만은 `slide-1.jpg`, 10-99페이지는 `slide-01.jpg`, 100페이지 이상은 `slide-001.jpg`.

수정 후 특정 슬라이드만 재렌더링하려면:

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 의존성

- `pip install "markitdown[pptx]"` — 텍스트 추출
- `pip install Pillow` — 썸네일 그리드
- `npm install -g pptxgenjs` — 처음부터 생성
- LibreOffice (`soffice`) — PDF 변환 (샌드박스 환경용 `scripts/office/soffice.py`로 자동 구성)
- Poppler (`pdftoppm`) — PDF를 이미지로 변환
