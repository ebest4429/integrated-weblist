# /validate - 공유 전 분석 결과 검증

> 익숙하지 않은 플레이스홀더가 있거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

이해관계자와 공유하기 전에 분석의 정확성, 방법론, 잠재적 편향을 검토합니다. 신뢰도 평가와 개선 제안을 생성합니다.

## 사용법

```
/validate <검토할 분석 내용>
```

분석 대상은 다음과 같습니다:
- 대화 내 문서 또는 보고서
- 파일 (마크다운, 노트북, 스프레드시트)
- SQL 쿼리 및 결과
- 차트 및 기초 데이터
- 방법론 및 분석 결과 설명

## 워크플로우

### 1. 방법론 및 가정 검토

다음 항목을 점검합니다:

- **질문 구성**: 분석이 올바른 질문에 답하고 있는가? 질문이 다르게 해석될 수 있는가?
- **데이터 선택**: 적절한 테이블/데이터셋을 사용하고 있는가? 시간 범위가 적절한가?
- **모집단 정의**: 분석 모집단이 올바르게 정의되어 있는가? 의도하지 않은 제외가 있는가?
- **지표 정의**: 지표가 명확하고 일관되게 정의되어 있는가? 이해관계자가 이해하는 방식과 일치하는가?
- **기준선 및 비교**: 비교가 공정한가? 시간 기간, 코호트 크기, 맥락이 비교 가능한가?

### 2. 일반적인 분석 오류 점검

다음 항목을 체계적으로 검토합니다:

**데이터 완전성:**
- 결과를 왜곡할 수 있는 누락 데이터 (예: 핵심 필드의 null, 누락된 시간 기간)
- 데이터 최신성 문제 (가장 최근 데이터가 실제로 완전한가, 아직 로딩 중인가?)
- 생존자 편향 (분석 날짜까지 "생존한" 개체만 보고 있는가?)

**통계적 문제:**
- 심슨의 역설 (데이터 집계 vs. 세분화 시 추세가 역전됨)
- 지지 증거 없이 상관관계를 인과관계로 제시
- 신뢰할 수 없는 결론으로 이어지는 작은 표본 크기
- 평균에 과도하게 영향을 미치는 이상값 (대신 중앙값을 사용해야 하는가?)
- 다중 검정 / 유의한 결과만 선택적으로 취합

**집계 오류:**
- 부적절한 조인으로 인한 이중 계산 (다대다 폭발)
- 비율 계산에서 잘못된 분모
- 세부 수준 혼합 (예: 사용자 수준 지표를 계정 수준과 평균 산출)
- 매출 인식 vs. 청구 vs. 수금 혼동

**시간 관련 문제:**
- 비교에서 계절성 미반영
- 평균에 불완전한 기간 포함 (예: 전체 월 대비 부분 월 비교)
- 데이터 소스 간 시간대 불일치
- 미래 전망 편향 (과거 사건을 설명하기 위해 미래 정보 사용)

**선택 및 범위:**
- 특정 내러티브에 유리하게 선택된 시간 범위
- 근거 없이 제외된 세그먼트
- 분석 중간에 변경된 정의

### 3. 계산 및 집계 검증

가능한 경우 다음을 확인합니다:

- 주요 수치를 독립적으로 재계산
- 소계가 합계와 일치하는지 검증
- 백분율이 예상대로 100%에 가까운지 확인
- YoY/MoM 비교가 올바른 기준 기간을 사용하는지 확인
- 모든 지표에 필터가 일관되게 적용되는지 검증

### 4. 시각화 평가

분석에 차트가 포함된 경우:

- 축이 적절한 값에서 시작하는가 (막대 차트의 경우 0)?
- 비교 차트 간 척도가 일관적인가?
- 차트 제목이 내용을 정확하게 설명하는가?
- 빠르게 훑어보는 독자에게 시각화가 오해를 줄 수 있는가?
- 인식을 왜곡하는 잘린 축, 불일치한 간격, 3D 효과가 있는가?

### 5. 내러티브 및 결론 평가

다음을 검토합니다:

- 결론이 제시된 데이터로 뒷받침되는가
- 대안적 설명이 인정되는가
- 불확실성이 적절하게 전달되는가
- 권고사항이 분석 결과에서 논리적으로 도출되는가
- 신뢰 수준이 증거의 강도와 일치하는가

### 6. 개선사항 제안

구체적이고 실행 가능한 제안을 제공합니다:

- 결론을 강화할 추가 분석
- 명시해야 할 주의사항 또는 한계점
- 핵심 포인트를 위한 더 나은 시각화 또는 표현 방식
- 이해관계자가 원할 누락된 맥락

### 7. 신뢰도 평가 생성

3단계 척도로 분석을 평가합니다:

**공유 준비 완료** — 방법론적으로 건전하고, 계산이 검증되었으며, 주의사항이 명시됨. 개선을 위한 소소한 제안은 있지만 차단 사항 없음.

**주의사항 명시 후 공유 가능** — 분석은 대체로 정확하지만 이해관계자에게 반드시 전달해야 할 특정 한계점 또는 가정이 있음. 필수 주의사항 목록 제시.

**수정 필요** — 공유 전 수정이 필요한 특정 오류, 방법론적 문제 또는 누락된 분석 발견. 우선순위 순서와 함께 필요한 변경사항 목록 제시.

## 출력 형식

```
## 검증 보고서

### 전체 평가: [공유 준비 완료 | 주의사항 명시 후 공유 가능 | 수정 필요]

### 방법론 검토
[접근 방식, 데이터 선택, 정의에 관한 결과]

### 발견된 문제
1. [심각도: 높음/중간/낮음] [문제 설명 및 영향]
2. ...

### 계산 확인 결과
- [지표]: [검증됨 / 불일치 발견]
- ...

### 시각화 검토
[차트 또는 시각적 표현의 문제점]

### 개선 제안
1. [개선사항 및 중요한 이유]
2. ...

### 이해관계자를 위한 필수 주의사항
- [반드시 전달해야 할 주의사항]
- ...
```

## 예시

```
/validate 임원진에게 보내기 전에 이 분기 매출 분석을 검토해 주세요: [분석 내용]
```

```
/validate 이탈률 분석을 확인해 주세요 — Q4 이탈률을 Q3와 비교하고 있는데 Q4의 측정 기간이 더 짧습니다
```

```
/validate 우리 전환 퍼널에 대한 SQL 쿼리와 결과입니다. 로직이 맞는지 확인해 주세요: [쿼리 + 결과]
```

## 팁

- 중요도가 높은 프레젠테이션이나 의사결정 전에 /validate를 실행하세요
- 간단한 분석도 정상 확인을 통해 이점을 얻을 수 있습니다 — 1분이면 되고 신뢰도를 지킬 수 있습니다
- 검증에서 문제가 발견되면 수정 후 다시 검증하세요
- 이해관계자의 신뢰를 높이기 위해 분석 결과와 함께 검증 보고서를 공유하세요
