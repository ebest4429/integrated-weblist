# /create-viz - 시각화 생성

> 익숙하지 않은 플레이스홀더가 보이거나 어떤 도구가 연결되어 있는지 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

Python을 사용하여 출판 품질의 데이터 시각화를 생성합니다. 명확성, 정확성, 디자인의 모범 사례를 적용하여 데이터에서 차트를 생성합니다.

## 사용법

```
/create-viz <데이터 소스> [차트 유형] [추가 지시사항]
```

## 워크플로우

### 1. 요청 파악

다음 사항을 결정합니다:

- **데이터 소스**: 쿼리 결과, 붙여넣은 데이터, CSV/Excel 파일, 또는 쿼리가 필요한 데이터
- **차트 유형**: 명시적으로 요청되었거나 추천이 필요한 경우
- **목적**: 탐색, 발표, 보고서, 대시보드 구성요소
- **대상**: 기술 팀, 임원진, 외부 이해관계자

### 2. 데이터 가져오기

**데이터 웨어하우스가 연결되어 있고 쿼리가 필요한 경우:**
1. 쿼리 작성 및 실행
2. 결과를 pandas DataFrame으로 로드

**데이터를 붙여넣거나 업로드한 경우:**
1. 데이터를 pandas DataFrame으로 파싱
2. 필요에 따라 정제 및 준비 (타입 변환, null 처리)

**대화의 이전 분석에서 나온 데이터인 경우:**
1. 기존 데이터 참조

### 3. 차트 유형 선택

사용자가 차트 유형을 지정하지 않은 경우, 데이터와 질문을 바탕으로 추천합니다:

| 데이터 관계 | 추천 차트 |
|---|---|
| 시간에 따른 추세 | 꺾은선 차트 |
| 범주 간 비교 | 막대 차트 (범주가 많으면 수평 막대) |
| 전체 대비 부분 구성 | 누적 막대 또는 영역 차트 (파이 차트는 6개 미만 범주에서만 사용) |
| 값의 분포 | 히스토그램 또는 박스 플롯 |
| 두 변수 간 상관관계 | 산점도 |
| 시간에 따른 두 변수 비교 | 이중 축 꺾은선 또는 그룹 막대 |
| 지리 데이터 | 코로플레스 지도 |
| 순위 | 수평 막대 차트 |
| 흐름 또는 프로세스 | 생키 다이어그램 |
| 관계 매트릭스 | 히트맵 |

사용자가 지정하지 않은 경우 간략히 추천 이유를 설명합니다.

### 4. 시각화 생성

필요에 따라 다음 라이브러리 중 하나를 사용하여 Python 코드를 작성합니다:

- **matplotlib + seaborn**: 정적이고 출판 품질의 차트에 적합. 기본 선택.
- **plotly**: 인터랙티브 차트 또는 사용자가 인터랙티비티를 요청할 때 적합.

**코드 요구사항:**

```python
import matplotlib.pyplot as plt
import seaborn as sns
import pandas as pd

# 전문적인 스타일 설정
plt.style.use('seaborn-v0_8-whitegrid')
sns.set_palette("husl")

# 적절한 크기로 Figure 생성
fig, ax = plt.subplots(figsize=(10, 6))

# [차트별 코드]

# 항상 포함할 것:
ax.set_title('명확하고 설명적인 제목', fontsize=14, fontweight='bold')
ax.set_xlabel('X축 레이블', fontsize=11)
ax.set_ylabel('Y축 레이블', fontsize=11)

# 숫자 형식 적절히 지정
# - 퍼센트: '45.2%' (0.452 금지)
# - 통화: '$1.2M' (1200000 금지)
# - 큰 숫자: '2.3K' 또는 '1.5M' (2300 또는 1500000 금지)

# 불필요한 테두리 제거
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

plt.tight_layout()
plt.savefig('chart_name.png', dpi=150, bbox_inches='tight')
plt.show()
```

### 5. 디자인 모범 사례 적용

**색상:**
- 일관되고 색맹 친화적인 팔레트 사용
- 색상을 의미 있게 사용 (장식용 사용 금지)
- 대비 색상으로 핵심 데이터 포인트 또는 추세 강조
- 덜 중요한 참조 데이터는 회색으로 처리

**타이포그래피:**
- 단순 지표가 아닌 인사이트를 나타내는 설명적 제목 (예: "매출 월별 추이"가 아닌 "매출 전년 대비 23% 성장")
- 읽기 쉬운 축 레이블 (가능하면 90도 회전 금지)
- 명확성을 높이는 경우 주요 포인트에 데이터 레이블 표시

**레이아웃:**
- 적절한 여백과 마진
- 데이터를 가리지 않는 범례 배치
- 자연스러운 순서가 없는 경우 범주를 값 기준으로 정렬 (알파벳 순 금지)

**정확성:**
- 막대 차트의 Y축은 0에서 시작
- 명확한 표시 없이 오해를 유발하는 축 단절 금지
- 패널 비교 시 일관된 스케일 유지
- 적절한 정밀도 (소수점 10자리 표시 금지)

### 6. 저장 및 제시

1. 설명적인 이름으로 PNG 파일로 차트 저장
2. 사용자에게 차트 표시
3. 수정 가능하도록 사용된 코드 제공
4. 변형 제안 (다른 차트 유형, 다른 그룹화, 확대된 시간 범위)
