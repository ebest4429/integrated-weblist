# 플러그인 한글화 작업

대상 폴더: `$ARGUMENTS`

> 경로 해석 규칙:
> - 폴더명만 입력 시 → 현재 작업 디렉토리 기준 상대경로로 해석
>   예) `/translate-plugin brand-guidelines` → `{현재 디렉토리}/brand-guidelines`
> - 하위 경로 입력 시 → 현재 작업 디렉토리 기준으로 해석
>   예) `/translate-plugin src/components` → `{현재 디렉토리}/src/components`

## 사전 확인

1. 대상 폴더 내 **전체 파일 목록**을 확인한다 (하위 폴더 포함)
2. 각 파일을 읽어 **영문 내용 포함 여부**를 판단한다
3. 이미 한글로 완성된 파일은 건너뛴다

---

## 파일 유형별 작업 규칙

### `.md` 파일 (마크다운)
- 영문 내용이 있으면 **전체 한글 번역**
- 코드 블록 내부 코드는 번역하지 않음 (주석·설명 문자열은 번역)
- 파일 덮어쓰기 (원본 교체)

### `.py` 파일 (Python)
**두 가지 작업 모두 수행:**

1. **파일 상단에 한글 요약 가이드 주석 추가** (형식 아래 참조)
2. **모든 독스트링·docstring을 한글로 번역**
3. **print 문, 오류 메시지, argparse help 등 사용자 노출 문자열 한글화**
4. 코드 로직·변수명·함수명은 변경하지 않음
5. LLM에 직접 전달되는 프롬프트 문자열(예: EVALUATION_PROMPT)은 **영문 유지**, 바로 아래에 한글 번역을 `#` 주석으로 추가

```python
"""
[파일명] (파일.py)
====================
[역할] 이 파일이 하는 일 한 줄 설명

[주요 기능]
  - 기능 1
  - 기능 2

[사용법]
  python 파일.py [인수]
  예: python 파일.py --옵션 값

[주요 클래스/함수]
  - ClassName  : 역할 설명
  - func_name(): 역할 설명
"""
```

### `.ts` / `.js` 파일 (TypeScript/JavaScript)
**두 가지 작업 모두 수행:**

1. **파일 상단에 한글 요약 가이드 주석 추가** (형식 아래 참조)
2. **JSDoc 주석·설명 문자열 한글화**
3. 코드 로직·변수명·함수명은 변경하지 않음

```typescript
/**
 * [파일명] (파일.ts)
 * ====================
 * [역할] 이 파일이 하는 일 한 줄 설명
 *
 * [주요 기능]
 *   - 기능 1
 *   - 기능 2
 *
 * [사용법]
 *   import { ... } from './파일'
 */
```

### `.xml` 파일
- 주석(`<!-- -->`) 형태로 상단에 한글 사용법 추가
- 태그 내 영문 텍스트 콘텐츠 한글 번역
- 태그명·속성명은 변경하지 않음

### `LICENSE.txt` / `LICENSE`
- 원본 영문 파일은 **절대 수정하지 않음**
- 같은 폴더에 `LICENSE_KR.txt` 새로 생성 (Apache 2.0 등 표준 라이선스 한글 번역)
- 파일 하단에 "※ 비공식 한글 번역본, 법적 효력은 원문 우선" 명시

### `requirements.txt` / `package.json` / `tsconfig.json`
- 패키지명·설정 키는 번역하지 않음
- `description` 필드 등 자연어 설명만 번역
- 실질적으로 번역할 내용 없으면 건너뜀

### 이미 한글화된 `.md` 파일 (예: `사용가이드.md`)
- 건너뜀

## 읽기 전용 파일 처리

파일 권한이 `r--r--r--` (읽기 전용)인 경우:
```bash
chmod 644 [파일경로]
```
권한 변경 후 작업 진행

## 작업 완료 후 보고 형식

```
## [폴더명] 한글화 완료

### 처리된 파일
| 파일 | 작업 내용 |
|------|-----------|
| 파일명.py | 상단 가이드 주석 + 독스트링 번역 |
| 파일명.md | 전체 번역 |
| LICENSE_KR.txt | 신규 생성 |

### 건너뛴 파일
| 파일 | 이유 |
|------|------|
| 파일명.md | 이미 한글 완성 |
| requirements.txt | 번역 내용 없음 |
```

## 주의사항

- 번역은 자연스러운 한국어 기술 문서 문체로 작성
- 전문 용어(MCP, SDK, API, stdio, async 등)는 영문 유지
- 코드 예시 내 문자열 리터럴도 맥락상 사용자 노출 문자면 번역
- 한 파일 작업 완료 후 다음 파일로 진행 (중간 실패 시 완료된 것부터 보고)
