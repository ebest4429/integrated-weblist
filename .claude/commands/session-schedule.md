현재 세션으로부터 재사용 가능한 단축 작업을 생성합니다. 아래 단계를 따릅니다:

## 1. 세션 분석

세션 기록을 검토하여 사용자가 수행하거나 요청한 핵심 작업을 파악합니다. 이를 하나의 반복 가능한 목표로 정리합니다.

## 2. 프롬프트 초안 작성

이 프롬프트는 향후 자율 실행에 사용됩니다 — 완전히 독립적으로 실행 가능해야 합니다. 향후 실행 시 이 세션에 접근할 수 없으므로, "현재 대화", "위의 내용", 또는 임시 맥락을 절대 참조하지 않습니다.

설명에 포함할 내용:
- 명확한 목표 진술 (달성할 내용)
- 실행할 구체적인 단계
- 관련 파일 경로, URL, 저장소 또는 도구명
- 예상 출력 또는 성공 기준
- 사용자가 표현한 제약 사항이나 선호도

설명은 2인칭 명령형으로 작성합니다 ("받은 편지함을 확인하세요…", "테스트 스위트를 실행하세요…"). 다른 Claude 세션이 처음부터 실행할 수 있을 만큼 충분히 완결되게 작성하되, 간결하게 유지합니다.

## 3. 작업명(taskName) 선택

케밥 케이스(kebab-case)로 짧고 설명적인 이름을 선택합니다 (예: "daily-inbox-summary", "weekly-dep-audit", "format-pr-description").

## 4. 일정 결정

다음 중 하나를 선택합니다:
- **반복 실행** ("매일 아침", "평일 오후 5시", "매시간") → `cronExpression`
- **특정 시점 1회 실행** ("5분 후 알림", "내일 오후 3시", "다음 주 금요일") → `fireAt` ISO 타임스탬프
- **임시(Ad-hoc)** (자동 실행 없음; 사용자가 수동으로 트리거) → 둘 다 생략
- **불명확** → 일정을 제안하고 진행 전 사용자 확인 요청

**cronExpression:** UTC가 아닌 사용자의 **로컬 시간대**를 기준으로 평가됩니다. 로컬 시간을 직접 사용합니다 — 예: "매주 금요일 오전 8시" → `0 8 * * 5`.

**fireAt:** 정확한 시점을 계산하여 시간대 오프셋이 포함된 전체 ISO 8601 문자열을 출력합니다 (예: `2026-03-05T14:30:00+09:00`). 1회 실행 작업에는 cron을 사용하지 않습니다 — cron은 1회 실행 의미론을 지원하지 않습니다.

마지막으로 "create_scheduled_task" 도구를 호출합니다.
