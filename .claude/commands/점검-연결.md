# /점검-연결 — 유기적 연결 점검

설정 파일들 간의 경로 참조, 포인터, 연결 상태가 실제 파일 구조와 일치하는지 점검한다.
구현 내용이 아닌 **설정/문서 레이어**의 내부 일관성을 검증한다.

---

## 실행 절차

### 0단계: scope 사전 검증

`.claude/점검-scope.md`를 읽고 각 섹션에 나열된 파일(주석 처리된 줄 제외)이 실제로 존재하는지 확인한다.

- 존재하지 않는 파일 → **이전 프로젝트 잔재 의심** 으로 분류
- 잔재 의심 항목이 있으면 목록을 출력하고 사용자에게 정리 여부를 확인한다
  - 정리 선택 → scope에서 해당 항목 제거 후 점검 진행
  - 무시 선택 → 해당 항목 제외하고 점검 진행

### 1단계: 범위 파악

`.claude/점검-scope.md`를 읽어 점검 대상 파일 목록을 확인한다.

### 2단계: 파일 전체 읽기

scope에 지정된 파일을 전부 읽는다.
- 기본 점검 대상 (고정)
- 프로젝트 추가 대상 (설정된 경우)
- 폴더로 지정된 경우 해당 폴더의 파일 전체를 읽는다

### 3단계: 연결 점검

아래 항목을 순서대로 점검한다.

#### A. 파일 존재 확인
- settings.json에 등록된 훅 파일 경로 → 실제 .js 파일 존재?
- CLAUDE.md에서 참조하는 파일 경로 → 실제 파일 존재?
- HOOK_GUIDE.md에 목록된 훅 → 실제 훅 파일 존재?
- SKILL_GUIDE.md에 목록된 스킬 → 실제 스킬 파일 존재?

#### B. 포인터 일치
- 마스터플랜 `**현재 활성 플랜:**` 포인터 → 해당 파일 실제 존재?
- 마스터플랜 활성 플랜 파일명 ↔ WORKSPACE.md `현재 플랜` 항목 일치?
- WORKSPACE.md Phase 상태 ↔ 마스터플랜 Phase 체크리스트 상태 일치?

#### C. 훅 구조 일치
- settings.json PreToolUse/PostToolUse 훅 목록 ↔ HOOK_GUIDE.md 목록 일치?
- hooks-guide/*.md 파일 수 ↔ settings.json 등록 훅 수 일치?
- 각 훅 가이드 파일명 ↔ settings.json 등록 훅 파일명 1:1 대응?

#### D. 경로 정합성
- CLAUDE.md, CONTEXT.md, RULES.md에서 참조하는 모든 파일 경로 → 실제 존재?
- settings.json SessionStart 훅 명령의 경로 참조 → 실제 존재?

### 4단계: 결과 정리

점검 결과를 다음 형식으로 정리한다.

```
## 점검 결과 — 유기적 연결
점검 일시: YYYY-MM-DD
점검 파일 수: N개

## 불일치 항목
| 번호 | 발견 파일 | 항목 | 문제 내용 | 심각도 |
|------|---------|------|---------|--------|

## 정상 확인 항목
- 항목 목록

## 수정 제안
각 불일치 항목별 수정 방향 (1줄 요약)
```

심각도 기준:
- 높음: 훅/스킬이 실제로 동작하지 않거나 차단되는 문제
- 중간: 포인터 불일치로 다음 세션 혼란 유발 가능
- 낮음: 가이드 문서와 실제 파일 간 불일치 (동작에 영향 없음)

### 5단계: 결과 저장 및 수정

1. 결과를 `.claude/점검결과/연결-YYYYMMDD.md`에 저장한다
   - 폴더가 없으면 생성한다
2. 불일치 항목 목록을 사용자에게 출력한다
3. 수정 진행 여부를 사용자에게 확인한다
4. 사용자가 선택한 항목만 수정한다 (전체 일괄 수정은 명시적 지시 후에만)
