# /analyze - 데이터 질문 답변

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인해야 하는 경우 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

빠른 조회부터 심층 분석, 공식 보고서까지 데이터 질문에 답변합니다.

## 사용법

```
/analyze <자연어 질문>
```

## 워크플로우

### 1. 질문 이해

사용자의 질문을 파악하고 다음 사항을 결정합니다:

- **복잡도 수준**:
  - **빠른 답변**: 단일 지표, 단순 필터, 사실 조회 (예: "지난 주에 가입한 사용자 수는?")
  - **전체 분석**: 다차원 탐색, 추세 분석, 비교 (예: "전환율이 떨어진 원인은?")
  - **공식 보고서**: 방법론, 주의사항, 권고사항을 포함한 종합 조사 (예: "구독 지표 분기 비즈니스 리뷰 작성")
- **데이터 요구사항**: 필요한 테이블, 지표, 차원, 기간
- **출력 형식**: 숫자, 표, 차트, 서술형 또는 조합

### 2. 데이터 수집

**데이터 웨어하우스 MCP 서버가 연결된 경우:**

1. 스키마를 탐색하여 관련 테이블과 컬럼을 찾습니다
2. 필요한 데이터를 추출하는 SQL 쿼리를 작성합니다
3. 쿼리를 실행하고 결과를 가져옵니다
4. 쿼리가 실패하면 디버그 후 재시도합니다 (컬럼명, 테이블 참조, 특정 방언의 문법 확인)
5. 결과가 예상과 다르면 진행 전 정합성 검사를 실행합니다

**데이터 웨어하우스가 연결되지 않은 경우:**

1. 다음 방법 중 하나로 데이터를 제공하도록 요청합니다:
   - 쿼리 결과 직접 붙여넣기
   - CSV 또는 Excel 파일 업로드
   - 스키마 설명 후 직접 실행할 쿼리 제공
2. 수동 실행용 쿼리를 작성하는 경우 `sql-queries` 스킬을 사용하여 방언별 모범 사례 적용
3. 데이터가 제공되면 분석을 진행합니다

### 3. 분석

- 관련 지표, 집계값, 비교값을 계산합니다
- 패턴, 추세, 이상값, 특이점을 식별합니다
- 차원 간 비교를 수행합니다 (기간, 세그먼트, 카테고리)
- 복잡한 분석의 경우 문제를 하위 질문으로 분해하여 각각 처리합니다

### 4. 결과 제시 전 검증

결과를 공유하기 전에 다음 검증 항목을 확인합니다:

- **행 수 정합성**: 레코드 수가 적절한가?
- **Null 확인**: 결과를 왜곡할 수 있는 예상치 못한 null이 있는가?
- **크기 확인**: 숫자가 합리적인 범위에 있는가?
- **추세 연속성**: 시계열에 예상치 못한 공백이 있는가?
- **집계 논리**: 소계가 합계와 올바르게 합산되는가?

확인 중 문제가 발견되면 조사하고 주의사항으로 기록합니다.

### 5. 결과 제시

**빠른 답변의 경우:**
- 관련 맥락과 함께 답변을 바로 제시합니다
- 재현성을 위해 사용된 쿼리를 포함합니다 (접힌 형태 또는 코드 블록)

**전체 분석의 경우:**
- 핵심 발견 또는 인사이트로 시작합니다
- 데이터 표 및/또는 시각화로 뒷받침합니다
- 방법론과 주의사항을 기록합니다
- 후속 질문을 제안합니다

**공식 보고서의 경우:**
- 핵심 요점이 담긴 요약(Executive Summary)
- 접근 방식과 데이터 소스를 설명하는 방법론 섹션
- 근거 자료가 포함된 상세 결과
- 주의사항, 한계, 데이터 품질 메모
- 권고사항 및 다음 단계 제안

### 6. 필요 시 시각화

표보다 차트가 결과를 더 효과적으로 전달하는 경우:

- `data-visualization` 스킬로 적합한 차트 유형을 선택합니다
- Python 시각화를 생성하거나 HTML 대시보드로 구성합니다
- 명확성과 정확성을 위한 시각화 모범 사례를 따릅니다

## 예시

**빠른 답변:**
```
/analyze 12월에 가입한 신규 사용자는 몇 명인가요?
```

**전체 분석:**
```
/analyze 지난 3개월간 지원 티켓 수가 증가한 원인은 무엇인가요? 카테고리와 우선순위별로 분류해 주세요.
```

**공식 보고서:**
```
/analyze 고객 테이블의 데이터 품질 평가를 작성해 주세요 -- 완전성, 일관성, 그리고 해결해야 할 문제들을 포함해서요.
```

## 팁

- 가능하면 기간, 세그먼트, 지표를 구체적으로 명시하세요
- 테이블 이름을 알고 있다면 언급하면 더 빠르게 처리됩니다
- 복잡한 질문은 여러 쿼리로 분해될 수 있습니다
- 결과는 항상 제시 전에 검증됩니다 — 이상한 점이 발견되면 표시됩니다
