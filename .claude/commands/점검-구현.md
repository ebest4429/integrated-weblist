# /점검-구현 — 구현 일치 점검

플랜에 명시된 항목이 실제 파일에 반영되었는지 점검한다.
설정 레이어가 아닌 **플랜 ↔ 실제 구현** 간의 갭을 검증한다.

---

## 실행 절차

### 1단계: 범위 파악

`.claude/점검-scope.md`를 읽어 점검 대상을 확인한다.
- `## 프로젝트 추가 대상` 의 플랜 폴더/파일 목록
- `## 구현 점검 대상` 의 실제 구현 파일 목록

### 2단계: 플랜 파일 읽기

scope에 지정된 플랜 파일을 전부 읽는다.
- 마스터플랜 — Phase 체크리스트 + 완료 조건
- 활성 페이지플랜 — `## 작업 항목` 체크박스 전체

읽으면서 아래를 추출한다:
- ✅ 완료 처리된 항목 목록
- 🔲 미완료 항목 목록
- 각 항목에 연결된 구현 스펙 (있는 경우)

### 3단계: 구현 파일 읽기

scope `## 구현 점검 대상`에 지정된 실제 파일을 전부 읽는다.
파일이 지정되지 않은 경우 사용자에게 확인 후 진행한다.

### 4단계: 구현 점검

#### A. 완료 항목 검증 (✅ 항목)
- 플랜에서 ✅로 표시된 항목 → 실제 파일에 해당 내용이 존재하는지 확인
- 플랜 스펙과 실제 파일 내용의 주요 차이점 파악

#### B. 미완료 항목 확인 (🔲 항목)
- 플랜에서 🔲인 항목 → 실제 파일에 부분적으로라도 구현되어 있는지 확인
- 이미 구현되었으나 플랜이 업데이트되지 않은 경우 식별

#### C. 스펙 일치 확인
- 플랜에 구체적인 스펙(화면 구조, 처리 로직 등)이 명시된 경우
- 해당 스펙과 실제 구현 내용의 일치 여부 확인

### 5단계: 결과 정리

점검 결과를 다음 형식으로 정리한다.

```
## 점검 결과 — 구현 일치
점검 일시: YYYY-MM-DD
점검 플랜: N개 / 구현 파일: N개

## 갭 항목
| 번호 | 플랜 항목 | 상태 | 문제 내용 |
|------|---------|------|---------|

## 정상 확인 항목
- 항목 목록 (플랜 ↔ 구현 일치 확인된 것)

## 플랜 업데이트 필요
- 이미 구현되었으나 플랜이 🔲인 항목 목록

## 수정 제안
각 갭 항목별 처리 방향 (구현 수정 또는 플랜 수정)
```

### 6단계: 결과 저장 및 수정

1. 결과를 `.claude/점검결과/구현-YYYYMMDD.md`에 저장한다
   - 폴더가 없으면 생성한다
2. 갭 항목 목록을 사용자에게 출력한다
3. 수정 진행 여부를 사용자에게 확인한다
4. 사용자가 선택한 항목만 수정한다 (전체 일괄 수정은 명시적 지시 후에만)

---

## 주의

- 구현 파일이 scope에 지정되지 않으면 점검 불가 — 점검 전 반드시 scope 파일에 추가
- 플랜 스펙이 추상적인 경우 Claude 판단으로 일치 여부를 결정하며, 불확실하면 사용자에게 확인
