# /write-query - 최적화된 SQL 쿼리 작성

> 익숙하지 않은 플레이스홀더가 있거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

자연어 설명으로부터 SQL 쿼리를 작성합니다. 특정 SQL 방언에 맞게 최적화되고 모범 사례를 따릅니다.

## 사용법

```
/write-query <필요한 데이터에 대한 설명>
```

## 워크플로우

### 1. 요청 파악

사용자의 설명을 분석하여 다음을 식별합니다:

- **출력 컬럼**: 결과에 어떤 필드가 포함되어야 하는가?
- **필터**: 데이터를 제한하는 조건은 무엇인가 (시간 범위, 세그먼트, 상태)?
- **집계**: GROUP BY 연산, 카운트, 합계, 평균이 필요한가?
- **조인**: 여러 테이블을 결합해야 하는가?
- **정렬**: 결과를 어떻게 정렬해야 하는가?
- **제한**: 상위 N개 또는 샘플 요건이 있는가?

### 2. SQL 방언 결정

사용자의 SQL 방언을 아직 모르는 경우 어떤 것을 사용하는지 질문합니다:

- **PostgreSQL** (Aurora, RDS, Supabase, Neon 포함)
- **Snowflake**
- **BigQuery** (Google Cloud)
- **Redshift** (Amazon)
- **Databricks SQL**
- **MySQL** (Aurora MySQL, PlanetScale 포함)
- **SQL Server** (Microsoft)
- **DuckDB**
- **SQLite**
- **기타** (구체적인 내용 질문)

같은 세션에서 이후 쿼리를 위해 방언을 기억합니다.

### 3. 스키마 탐색 (웨어하우스 연결 시)

데이터 웨어하우스 MCP 서버가 연결된 경우:

1. 사용자 설명을 바탕으로 관련 테이블 검색
2. 컬럼명, 타입, 관계 확인
3. 성능에 영향을 주는 파티셔닝 또는 클러스터링 키 확인
4. 쿼리를 단순화할 수 있는 미리 빌드된 뷰 또는 구체화 뷰 탐색

### 4. 쿼리 작성

다음 모범 사례를 따릅니다:

**구조:**
- 여러 논리적 단계가 있는 쿼리는 가독성을 위해 CTE(WITH 절) 사용
- 논리적 변환 또는 데이터 소스마다 하나의 CTE
- CTE에 서술적인 이름 부여 (예: `daily_signups`, `active_users`, `revenue_by_product`)

**성능:**
- 운영 쿼리에서 `SELECT *` 절대 사용 금지 — 필요한 컬럼만 지정
- 조기 필터링 (WHERE 절을 기본 테이블에 최대한 가깝게 배치)
- 가능한 경우 파티션 필터 사용 (특히 날짜 파티션)
- 결과 집합이 큰 서브쿼리에는 `IN` 대신 `EXISTS` 사용
- 적절한 JOIN 타입 사용 (INNER JOIN이 맞는 경우 LEFT JOIN 사용 금지)
- JOIN 또는 윈도우 함수로 처리 가능한 경우 상관 서브쿼리 지양
- 폭발적 조인(다대다) 주의

**가독성:**
- 명확하지 않은 로직에 "왜"를 설명하는 주석 추가
- 일관된 들여쓰기와 포맷 사용
- 테이블에 의미 있는 짧은 별칭 사용 (`a`, `b`, `c`만 사용 금지)
- 각 주요 절을 별도 줄에 배치

**방언별 최적화:**
- 방언별 문법과 함수 적용 (자세한 내용은 `sql-queries` 스킬 참조)
- 방언에 맞는 날짜 함수, 문자열 함수, 윈도우 문법 사용
- 방언별 성능 기능 명시 (예: Snowflake 클러스터링, BigQuery 파티셔닝)

### 5. 쿼리 제시

다음을 제공합니다:

1. **완전한 쿼리** — 구문 강조 표시가 된 SQL 코드 블록
2. **간단한 설명** — 각 CTE 또는 섹션이 수행하는 작업
3. **성능 참고사항** — 해당하는 경우 (예상 비용, 파티션 사용, 잠재적 병목 지점)
4. **수정 제안** — 일반적인 변형에 맞게 조정하는 방법 (다른 시간 범위, 다른 세부 수준, 추가 필터)

### 6. 실행 제안

데이터 웨어하우스가 연결된 경우 쿼리를 실행하고 결과를 분석하겠다고 제안합니다. 사용자가 직접 실행하길 원하면 쿼리를 바로 복사해서 사용할 수 있습니다.

## 예시

**단순 집계:**
```
/write-query 최근 30일간 상태별 주문 수
```

**복잡한 분석:**
```
/write-query 코호트 리텐션 분석 — 가입 월별로 사용자를 그룹화한 후, 가입 후 1, 3, 6, 12개월에 여전히 활성(최소 1회 이벤트 발생) 상태인 비율을 표시
```

**성능 중요 쿼리:**
```
/write-query 날짜별로 파티셔닝된 5억 행 이벤트 테이블이 있습니다. 최근 7일간 이벤트 수 기준 상위 100명의 사용자와 각자의 가장 최근 이벤트 유형을 조회해 주세요.
```

## 팁

- 올바른 문법을 즉시 얻으려면 SQL 방언을 미리 언급하세요
- 테이블명을 알고 있으면 포함하세요 — 그렇지 않으면 찾는 것을 도와드립니다
- 쿼리를 멱등적으로(재실행 안전) 만들어야 하는지, 1회성인지 명시하세요
- 반복 실행되는 쿼리의 경우 날짜 범위에 대해 파라미터화해야 하는지 언급하세요
