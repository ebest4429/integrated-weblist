# 재정계획 스킬 v1.0

지자체 중기지방재정계획 분석용 프롬프트 2종을 생성하는 챗봇형 스킬.

---

## 실행 환경

Claude Code CLI 전용. 파일은 직접 생성한다. ZIP 금지.

---

## 중요 원칙

- 한 번에 하나의 단계만 진행한다
- 각 단계에서 사용자 답변을 받은 후 다음 단계로 넘어간다
- 수집된 정보는 내부적으로 누적하여 최종 파일 생성에 사용한다
- 모든 텍스트는 순한글로 작성한다 — 析, 分 문자 사용 금지, '분석'은 반드시 순한글로만 작성한다
- 어느 단계에서든 '취소' 입력 시 "스킬을 종료합니다." 출력 후 즉시 종료한다
- Step 2에서 번호 목록에 없는 텍스트 입력은 직접입력 목적으로 처리한다
- Step 4에서 추천 분야는 참고용 표시만 하며 선택 상태로 자동 설정하지 않는다
- Step 4에서 선택 상태 변경(추가/제거/Y) 즉시 현재 선택 목록을 출력한다

---

## Step 0: 시작 안내

트리거 실행 시 다음을 출력한다:

```
지자체 중기지방재정계획 분석 프롬프트 생성을 시작합니다.

5단계 질문을 통해 분석용 파일 2종을 생성합니다.
  - NotebookLM_프롬프트.md  → NotebookLM에 투입하여 데이터 추출
  - Claude_세션1_시작프롬프트.md → 새 Claude 세션에서 분석 시작

준비되셨으면 첫 번째 질문입니다.
진행 중 취소하려면 '취소'를 입력하세요.
```

## Step 1: 지자체명 수집

```
[1/5] 분석할 지자체를 입력해주세요.

광역시·도 전체부터 기초단체(구·시·군)까지 원하는 단위로 입력하세요.

예) 대전광역시 / 대전광역시 서구 / 경기도 화성시 / 충청북도 옥천군

→ 지자체:
```

**처리:**

- 입력값 그대로 `지자체명`으로 저장
- 약칭 자동 생성 규칙:
  - 광역시·도 전체: 시도명 앞 2자 (대전광역시→대전, 경기도→경기, 충청북도→충북)
  - 기초단체 포함: 시도 앞 2자 + 기초단체명 (경기도 화성시→경기화성, 대전광역시 서구→대전서구, 충청북도 옥천군→충북옥천)

---

## Step 2: 분석 목적 수집 및 분야 추천

```
[2/5] 분석 목적을 선택해주세요. (복수 선택 가능, 번호 입력 후 엔터)
직접 입력도 가능합니다.

1. 부동산 투자 판단
2. 개발 방향 예측
3. 복지 정책 연구
4. 교육 인프라 파악
5. 관광·문화 분야 파악
6. 치안·안전 정책
7. 산업·경제 분석
8. 환경·녹지 계획
9. 직접 입력

선택 완료는 '0' 입력
→ 선택:
```

**처리:**

- 복수 선택 누적, '0' 입력 시 완료
- '9' 선택 시 직접 입력 받아 목적으로 저장
- 입력된 목적을 바탕으로 집중 분야를 자동 추천한다 (Step 4에서 제시)

**목적 → 분야 추천 매핑:**

| 목적 키워드 | 추천 분야 |
|------------|---------|
| 부동산, 투자, 개발 | 교통 및 물류, 국토 및 지역개발, 환경 |
| 복지, 의료, 노인 | 사회복지, 보건 |
| 교육, 학교 | 교육 |
| 관광, 문화 | 문화 및 관광 |
| 치안, 안전, 소방 | 공공질서 및 안전 |
| 산업, 경제, 중소기업 | 산업·중소기업 및 에너지 |
| 환경, 녹지, 공원 | 환경 |
| 농업, 수산 | 농림해양수산 |
| 과학기술, 연구 | 과학기술 |
| 행정, 일반 | 일반공공행정 |

## Step 3: 보유 데이터 확인

```
[3/5] NotebookLM에 업로드할 자료를 확인합니다.
보유하신 자료에 해당하는 번호를 모두 입력해주세요. (예: 1,2,4)

1. 재정 분석 자료 (연도별 PDF)
2. 투자사업 심사결과
3. 장기미집행 도시계획시설 집행계획
4. 도시기본계획 / 도시관리계획
5. 기타 재정 관련 자료
   (예: 자치구별 세부계획, 특정 분야 보고서 등)

→ 보유 자료 번호:
```

**처리:**

- 선택 항목을 `보유데이터목록`으로 저장
- 1번 미선택 시: "재정 분석 자료는 핵심 자료입니다. 보유 여부를 다시 확인해주세요." 안내

## Step 4: 집중 분야 확정

Step 2에서 추천된 분야를 번호와 함께 참고용으로 표시한다. 추천은 선택 상태가 아니다.

```
[4/5] 집중 분석 분야를 확정합니다.

분석 목적 기반 추천 분야 (참고):
[번호 포함하여 추천 분야 나열]

추천을 그대로 사용하려면 'Y'를 입력하세요.
번호로 직접 선택할 수도 있습니다. (추가: 번호 / 제거: -번호)
전체 목록은 'ㄷ', 선택 완료는 '0'.

현재 선택: (없음)
→ 선택:
```

'Y' 입력 시: 추천 분야 전체를 현재 선택에 추가하고 즉시 현재 선택 목록을 출력한다.

사용자가 'ㄷ'를 입력하면 전체 분야 목록 출력:

```
── 전체 분야 목록 (중기지방재정계획 대분류 기준) ──

1. 일반공공행정
2. 공공질서 및 안전
3. 교육
4. 문화 및 관광
5. 환경
6. 사회복지
7. 보건
8. 농림해양수산
9. 산업·중소기업 및 에너지
10. 교통 및 물류
11. 국토 및 지역개발
12. 과학기술

현재 선택: [누적 목록]
→ 선택 (번호 추가 / -번호 제거 / 'Y' 추천 전체 / '0' 완료):
```

사용자가 '0'을 입력하면:

```
최종 선택 분야: [전체 누적 목록]

이 분야들을 중심으로 파일을 생성합니다.
```

**처리:**

- 추천 분야는 참고용 표시만 하며 선택 상태로 자동 설정하지 않는다
- 추천 분야는 12개 대분류 번호 오름차순으로 정렬하여 표시한다
- 선택 상태 변경(추가/제거/Y) 즉시 현재 선택 목록을 출력한다
- 최종 선택 분야를 `집중분야목록`으로 저장

## Step 5: 분석 기간 설정

```
[5/5] 분석 기간을 설정합니다.

기본값: 2019 ~ 2028 (10년)
다른 기간이면 입력해주세요. 기본값이면 '0'을 입력하세요.

→ 분석 기간 (예: 2020-2028):
```

**처리:**

- '0' 또는 공백: 시작=2019, 종료=2028
- 입력값 있으면 해당 연도로 저장

## 파일 생성 로직

### 저장 위치 결정

프로젝트 루트에서 기존 폴더를 확인하여 순번을 자동 부여한다:

```
재정계획-[지자체명약칭]1/ 이 없으면 → 재정계획-[약칭]1/
재정계획-[약칭]1/ 이 있으면 → 재정계획-[약칭]2/
...
```

폴더를 생성하고 그 안에 파일 2종을 저장한다.

### 파일 1: NotebookLM\_프롬프트.md

**설계 원칙:** 분야명(대분류) 자체로 추출 요청. 키워드 하드코딩 없음.
지자체마다 세부사업명이 달라도 분야명 기반이면 빠짐없이 추출 가능.

**고정 서두:**

```
당신은 [지자체명] 재정 관련 문서들을 보유하고 있습니다.
아래 요청사항에 따라 데이터만 추출해주세요.
해석·분석·평가는 하지 마세요. 원본 데이터를 정리된 형태로만 제공해주세요.
사업명과 사업개요가 있는 모든 세부사업을 빠짐없이 추출해주세요.

보유 문서: [보유데이터목록]
분석 기간: [시작] ~ [종료]
```

**분야별 추출 섹션 (선택된 분야만, 12개 대분류 번호 순으로 생성):**

각 선택 분야에 대해 다음 형식으로 섹션 생성:

```
## 추출 요청 N: [분야명]

[분야명] 분야에 속하는 모든 세부사업을 추출해주세요.
사업개요가 존재하는 행(실질적 세부사업)만 포함합니다.

추출 항목 (표 형식):
| 연도 | 분야 | 부문 | 사업명 | 사업개요 | 예산(백만원) |

주의:
- [분야명] 분야로 분류된 모든 사업 포함 (사업명으로 임의 필터링 금지)
- 원문 그대로 출력 (요약/편집 금지)
- 동일 사업명 중복 항목도 전부 나열
```

보유 데이터 2번(투자심사결과) 선택 시: 아래 섹션 추가

```
## 추출 요청 [N+1]: 투자심사 통과 사업

집중 분야([집중분야목록]) 관련 투자심사 통과 사업 목록을 추출해주세요.

추출 항목 (표 형식):
| 연도 | 사업명 | 심사결과 | 총사업비(백만원) |

주의: 원문 그대로 출력 (요약/편집 금지)
```

보유 데이터 3번(장기미집행) 선택 시: 아래 섹션 추가

```
## 추출 요청 [N+2]: 장기미집행 도시계획시설

집중 분야([집중분야목록]) 관련 장기미집행 시설을 추출해주세요.

추출 항목 (표 형식):
| 시설명 | 분야 | 결정연도 | 집행예정연도 | 사업비(백만원) |

주의: 원문 그대로 출력 (요약/편집 금지)
```

**고정 마무리:**

```
## 출력 기준

- 마크다운 표 형식으로 정리
- 데이터가 없는 연도는 "해당 없음"으로 표시
- 사업명이 불명확한 경우 원문 그대로 기재
- 해석·분석·의견은 절대 추가하지 않음
```

### 파일 2: Claude\_세션1\_시작프롬프트.md

**설계 원칙:** 선택 분야 수만큼 Phase 자동 구성. 마지막 Phase는 종합 인사이트.

```
# [지자체명] 재정계획 분석 — 세션 1 시작

## 분석 목적
[분석목적 (복수 선택 시 목록 형태)]

## 분석 기간
[시작] ~ [종료]

## 보유 데이터
[보유데이터목록]

## 집중 분야
[집중분야목록]

---

## 전체 분석 로드맵

NotebookLM에서 추출한 데이터를 기반으로 아래 순서로 분석을 진행합니다.

Phase 1: [첫 번째 선택 분야] 분석
Phase 2: [두 번째 선택 분야] 분석
... (선택 분야 수만큼 자동 생성)
Phase [N+1]: 종합 인사이트 도출 — [분석목적] 관점

---

## 세션 1 분석 요청

[첫 번째 선택 분야] 관련 데이터를 붙여넣어 주세요.
데이터를 받으면 다음을 분석합니다:

1. 투자 규모 및 연도별 추이
2. 주요 사업 목록 및 특징
3. [분석목적] 관점에서의 시사점

---
*이 프롬프트는 /재정계획 스킬로 자동 생성되었습니다.*
*생성 일시: [생성날짜]*
```
